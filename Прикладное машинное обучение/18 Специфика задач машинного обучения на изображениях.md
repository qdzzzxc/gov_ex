# Специфика задач машинного обучения на изображениях. Принцип работы сверточных искусственных нейронных сетей. Преимущества сверточных сетей при решении задач на изображениях.

## TL;DR
Изображения — высокоразмерные тензоры $H\times W\times C$ с локальными корреляциями и желаемыми устойчивостями к сдвигу, масштабу и малым поворотам. Полносвязные сети плохо масштабируются (взрыв параметров). **Свёрточные сети (CNN)** используют **локальные фильтры** (свёртки), **разделение весов** (тот же фильтр по всему изображению), **пулинг** (downsampling). Свёрточный слой: $z(i,j) = \sum_{u,v,c} W(u,v,c) x(i+u, j+v, c) + b$. Преимущества: меньше параметров, эквивариантность свёртки к сдвигам и частичная инвариантность за счёт пулинга/агрегации, иерархичность признаков (от линий до объектов). Современные архитектуры: ResNet, EfficientNet, Vision Transformers.

## Развёрнуто

### Специфика задач на изображениях
- **Высокая размерность**: 224×224×3 = 150528 пикселей; полносвязный слой к 1000 нейронам = 150 млн параметров.
- **Пространственная корреляция**: соседние пиксели коррелированы; информативны локальные паттерны.
- **Иерархия признаков**: грани → углы → текстуры → части → объекты.
- **Инвариантности**: сдвиг (объект где угодно), малые повороты, масштаб, яркость.
- **Большой объём данных**: сотни тысяч–миллионы примеров; нужен GPU/TPU.

### Типы задач
- **Классификация**: одна метка на изображение (ImageNet).
- **Локализация**: bounding box объекта.
- **Детекция**: множество объектов с боксами и классами (YOLO, Faster R-CNN, DETR).
- **Сегментация**: попиксельная классификация (semantic / instance / panoptic). U-Net, Mask R-CNN.
- **Generative**: GAN, диффузионные модели для генерации.
- **Style transfer, super-resolution, depth estimation**.

### Свёрточный слой
Для входа $x\in\mathbb R^{H\times W\times C}$ и набора фильтров $W\in\mathbb R^{k\times k\times C\times F}$ ($k$ — размер ядра, $F$ — число выходных каналов):
$$z_{i,j,f} = \sum_{u=0}^{k-1}\sum_{v=0}^{k-1}\sum_{c=1}^C W_{u,v,c,f}\cdot x_{i+u, j+v, c} + b_f.$$
**Параметры**: stride $s$, padding $p$. Выходная размерность:
$$H_{out} = \lfloor (H + 2p - k)/s\rfloor + 1.$$

**Активация**: ReLU после свёртки (стандарт).

### Пулинг
Понижает разрешение карты признаков:
- **Max pooling** $2\times 2$: максимум в окне → инвариантность к малым сдвигам.
- **Average pooling**: среднее — мягче.
- **Global Average Pooling** — усреднение по всему пространству; используется перед классификатором (заменяет полносвязные слои, экономит параметры).

### Структура CNN
Типичная архитектура:
[Conv → ReLU → BatchNorm] × N → Pool → [Conv → ReLU → BN] × N → Pool → ... → GAP → Dense → Softmax.

**Современные блоки**:
- **Residual block (ResNet)**: $h = f(x) + x$ — пропускающее соединение. Позволяет обучать очень глубокие сети.
- **Inception** — параллельные ветви разных размеров ядра.
- **Depthwise separable convolutions** (MobileNet, EfficientNet) — экономия параметров.
- **Squeeze-and-Excitation** — внимание по каналам.
- **Vision Transformer (ViT)** — patch + self-attention, без свёрток.

### Преимущества CNN
1. **Разделение весов (weight sharing)**: один фильтр сканирует всё изображение → намного меньше параметров. Свёртка $3\times 3$ с $C=3, F=64$ имеет $3\cdot 3\cdot 3\cdot 64 = 1728$ параметров, независимо от $H\times W$.
2. **Локальные рецептивные поля**: свёртки опираются на локальные паттерны, что соответствует структуре изображения.
3. **Эквивариантность к сдвигу**: сдвиг входа сдвигает выход; пулинг даёт частичную **инвариантность**.
4. **Иерархическое представление**: ранние слои — низкоуровневые признаки, поздние — высокоуровневые.
5. **Эффективность на GPU**: свёртки — оптимизированные операции (cuDNN).

### Регуляризация и обучение
- **Data augmentation**: случайный crop, flip, цветовые изменения, mixup, cutmix — критично для обобщения.
- **Dropout**, **DropPath** в residual блоках.
- **BatchNorm** / **LayerNorm** — стабилизация обучения.
- **Transfer learning**: предобученная на ImageNet модель → дообучение на задаче. Стандартный приём.
- **Self-supervised pre-training**: SimCLR, MoCo, MAE, DINO.

### Пример: классификация цифр (MNIST)
```python
import torch.nn as nn
import torch
class CNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.pool = nn.MaxPool2d(2, 2)
        self.fc = nn.Linear(64*7*7, 10)
    def forward(self, x):
        x = self.pool(torch.relu(self.conv1(x)))
        x = self.pool(torch.relu(self.conv2(x)))
        x = x.view(x.size(0), -1)
        return self.fc(x)
```
Обучение: SGD/Adam + cross-entropy. На MNIST такая сеть даёт >99% accuracy.

### Подводные камни
- Без аугментации малые датасеты дают сильное переобучение.
- BatchNorm ведёт себя по-разному в train/eval — частая ошибка.
- Без residual connections глубокие CNN плохо обучаются (vanishing gradients).
- Smaller kernels with deeper nets — обычно лучше, чем крупные ядра.
- Vision Transformers требуют **много данных** или предобучения; CNN устойчивее на малых выборках.
