# Специфика задач машинного обучения на изображениях. Принцип работы сверточных искусственных нейронных сетей. Преимущества сверточных сетей при решении задач на изображениях.

**Специфика изображений:**
- Высокая размерность ($224 \times 224 \times 3 = 150528$ пикселей).
- Локальные пространственные корреляции.
- Иерархия признаков: границы → углы → текстуры → части → объекты.
- Инвариантности: сдвиг, малые повороты, масштаб.

**Почему не FC:** полносвязный слой $224 \times 224 \times 3 \to 1000$ нейронов = 150M параметров. Не масштабируется + не использует пространственную структуру.

### Свёрточный слой
$$z_{i,j,f} = \sum_{u,v,c} W_{u,v,c,f} \cdot x_{i+u, j+v, c} + b_f.$$

**Фильтр** — кубик $k \times k \times C_{in}$ весов. В слое $F$ таких кубиков → выход с $F$ каналами. Параметры: $k \cdot k \cdot C_{in} \cdot F$ + $F$ bias.

**Stride** ($s$) — шаг фильтра по входу. **Padding** ($p$) — нули по краям для покрытия краевых пикселей.

**Выходная размерность:** $H_{out} = \lfloor (H + 2p - k) / s \rfloor + 1$.

### Пулинг
Понижает разрешение карты признаков:
- **Max pool** — максимум в окне, даёт локальную инвариантность к сдвигам.
- **Average pool** — среднее, мягче.
- **Global Average Pooling (GAP)** — усреднение по всей пространственной размерности, заменяет финальные FC слои.

### Архитектура
Типично: `[Conv → BN → ReLU] × N → Pool → ... → GAP → Dense → Softmax`.

**Современные блоки:**
- **Residual (ResNet):** $y = f(x) + x$, skip connection. Решает vanishing gradient, делает loss-ландшафт глаже, позволяет обучать глубокие сети.
- **Inception** — параллельные ветви разных размеров ядра.
- **Depthwise separable convs (MobileNet)** — экономия параметров.
- **Vision Transformer** — без свёрток, патчи + self-attention.

### Преимущества CNN
1. **Разделение весов (weight sharing):** один фильтр сканирует всё изображение, его 27 весов "переиспользуются" в каждой позиции. Параметров **в тысячи раз меньше**, чем у FC при том же выходе. Работает потому, что локальные признаки (границы, текстуры) одинаковы во всех позициях изображения.
2. **Локальные рецептивные поля** — соответствуют структуре изображения.
3. **Эквивариантность к сдвигу** (свёртка сама): сдвинул вход → выход сдвигается так же.
4. **Инвариантность** (через пулинг и GAP): финальное предсказание не зависит от позиции объекта.
5. **Иерархия признаков** — низкоуровневые в ранних слоях, высокоуровневые в поздних. Эмерджентно возникает благодаря разной случайной инициализации фильтров (без неё все 64 канала остались бы клонами — symmetry breaking).
6. **Эффективность на GPU** — свёртки сильно оптимизированы (cuDNN).

### Эквивариантность vs инвариантность
- **Эквивариантность:** $\text{CNN}(\text{сдвиг}(x)) = \text{сдвиг}(\text{CNN}(x))$. Свойство свёртки.
- **Инвариантность:** $\text{CNN}(\text{сдвиг}(x)) = \text{CNN}(x)$. Получается через пулинг + GAP.

Для классификации нужна инвариантность ("что"). Для детекции/сегментации — эквивариантность ("где").
### Регуляризация и обучение
- **Data augmentation:** crop, flip, color jitter, mixup, cutmix — критично для обобщения.
- **BatchNorm / LayerNorm** — стабилизация.
- **Dropout / DropPath**.
- **Transfer learning** — предобученная на ImageNet модель + дообучение. Стандартный приём.
- **Self-supervised pre-training:** SimCLR, MoCo, MAE, DINO.
### Задачи
- Классификация (одна метка) — ResNet, EfficientNet, ViT.
- Детекция (объекты + боксы) — YOLO, Faster R-CNN, DETR.
- Сегментация (попиксельная) — U-Net, Mask R-CNN.
- Генерация — GAN, диффузионные модели.
### Подводные камни
- Без augmentation малые датасеты → переобучение.
- BatchNorm ведёт себя по-разному в `train`/`eval`.
- Глубокие сети без residual connections не обучаются (vanishing gradient).
- ViT требует много данных; CNN устойчивее на малых выборках.
### Главное на экзамене
1. **Свёртка vs FC** — экономия параметров через weight sharing.
2. **Структура свёрточного слоя** (фильтры, stride, padding, формула выхода).
3. **Пулинг** — зачем и виды.
4. **Преимущества CNN:** локальность, weight sharing, эквивариантность/инвариантность, иерархия.
5. **Residual connections** — что и зачем.
6. **Transfer learning** — что и зачем.
