# Организация вычислений с помощью Map / Filter / Reduce: общий принцип и специфика параллельной реализации обработки данных в Dask.Bag.

## Триада функциональной обработки
- **Map** $f: A \to B$: применяет функцию к каждому элементу. $\mathrm{map}(f, [x_1, \dots]) = [f(x_1), \dots]$. Полностью независимая обработка — идеально параллелится.
- **Filter** $p: A \to {T, F}$: оставляет элементы по предикату. Тоже тривиально параллелится.
- **Reduce / fold:** сворачивает коллекцию ассоциативным оператором $\oplus$ к одному значению.
## Ассоциативность — ключ к параллельному reduce
$(a \oplus b) \oplus c = a \oplus (b \oplus c)$ позволяет вычислять через сбалансированное дерево: глубина $O(\log n)$ вместо $O(n)$. Примеры ассоциативных операций: $+, \times, \min, \max$, конкатенация. Не ассоциативные: $-, /$.
## Параллельная реализация — две фазы
1. **Локальная фаза:** на каждом partition выполняем reduce внутри → один частичный результат на partition.
2. **Глобальная фаза:** собираем частичные результаты, сворачиваем финальным reduce. Преимущество: сетевой трафик минимальный — передаются только частичные результаты, не сырые данные.
## Чистая функция
Функция чистая, если: возвращает результат только на основе входов + не имеет побочных эффектов (не меняет глобальное состояние, не пишет в файл). Зачем для параллелизма: без чистоты — race condition при одновременном изменении общего состояния разными потоками. Дополнительно даёт оптимизации композиции: $\mathrm{map}(g) \circ \mathrm{map}(f) = \mathrm{map}(g \circ f)$ (fusion — один проход вместо двух).
## Dask.Bag
Ленивая параллельная коллекция произвольных Python-объектов. Аналог PySpark RDD для Python. Создание:
```python
b = db.from_sequence([1,2,3,4,5], npartitions=2)
b = db.read_text("logs/*.txt")
```
Основные методы:
- `b.map(f)` / `b.filter(p)` / `b.flatten()` / `b.distinct()`.
- `b.fold(binop, initial)` — reduce, нужна ассоциативность.
- `b.groupby(key)` — полный shuffle, дорого.
- `b.foldby(key, binop, initial)` — groupby + reduce с **локальной агрегацией** на partition'ах перед shuffle. Сильно эффективнее groupby + map.
- `b.frequencies()`, `b.topk(n, key)` — частые операции.
- `b.compute()` — запустить вычисление. Ленивые вычисления: операции строят task-граф, выполняются только при `.compute()`. Позволяют оптимизацию (fusion), экономию памяти, распределение по узлам.
