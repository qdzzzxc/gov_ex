# Классификация как задача машинного обучения. Постановка задачи, математическая формализация. Целевая функция. Примеры.

## TL;DR
**Задача классификации**: по объекту $x\in\mathcal X$ предсказать дискретную метку $y\in\{1, \dots, K\}$ (класс). Формально: оценить $f: \mathcal X\to\mathcal Y$ на выборке $\{(x_i, y_i)\}$, минимизируя ожидание лосса. **Целевые функции**: 0-1 ошибка (теоретическая), **кросс-энтропия** (logistic loss) для вероятностных моделей, **hinge loss** для SVM, **focal loss** для несбалансированных задач. Модели: логистическая регрессия, SVM, деревья и ансамбли (RF, бустинг), kNN, наивный байес, нейросети. Оптимальный байесовский классификатор: $f^*(x) = \arg\max_k P(y=k|x)$.

## Развёрнуто

### Постановка
Дано: обучающая выборка $\{(x_i, y_i)\}_{i=1}^n$, $x_i\in\mathcal X\subseteq\mathbb R^d$, $y_i\in\{1,\dots,K\}$.

Цель: найти классификатор $f: \mathcal X\to\{1,\dots,K\}$, чтобы для будущих $(x, y)\sim P$ ошибка $L(f(x), y)$ была мала в среднем.

Часто строят **скоринговую** модель $g: \mathcal X\to\mathbb R^K$ (или вероятностную $p(y|x)$), и $f(x) = \arg\max_k g_k(x)$.

### Виды задач классификации
- **Бинарная**: $K = 2$ (диагноз болезнь/здоровье, спам/не-спам).
- **Мультикласс** (one-vs-rest, softmax).
- **Мультилейбл**: каждому $x$ — подмножество меток.
- **Иерархическая** (деревья меток).
- **Imbalanced** (несбалансированные классы) — особый режим.

### Целевые функции
**0-1 ошибка** (теоретическая):
$$L_{0-1}(f(x), y) = \mathbb 1\{f(x)\ne y\}.$$
Эмпирический риск $\hat R = \tfrac{1}{n}\sum \mathbb 1\{f(x_i)\ne y_i\}$. Не дифференцируема — на практике заменяется суррогатом.

**Кросс-энтропия / log-loss** (для вероятностных моделей $p(y=k|x) = \pi_k$):
$$L_{CE} = -\sum_{k=1}^K \mathbb 1\{y=k\}\,\ln \pi_k.$$
Для бинарной: $L = -y\ln p - (1-y)\ln(1-p)$.

Эквивалентна максимизации правдоподобия. Применяется в логистической регрессии, нейросетях.

**Hinge loss** (SVM):
$$L_{H} = \max(0, 1 - y\cdot g(x)),\quad y\in\{-1, +1\}.$$
Геометрически — штраф за нахождение в зазоре.

**Focal loss** (Lin et al.) — для несбалансированных классов:
$$L_F = -(1-p_y)^\gamma\ln p_y.$$
Уменьшает вес легко классифицируемых примеров.

### Байесовский оптимальный классификатор
$$f^*(x) = \arg\max_k P(y=k|x).$$
По Байесу: $P(y=k|x) = P(x|y=k)P(y=k)/P(x)$.
Минимизирует $\mathbb E L_{0-1}$ — теоретическая нижняя граница ошибки (байесовская ошибка).

### Модели

**Логистическая регрессия (бинарная)**: $p(y=1|x) = \sigma(w^\top x + b)$, $\sigma(z) = 1/(1+e^{-z})$. Обучение — минимизация log-loss. Линейная разделяющая поверхность.

**Softmax-регрессия (multinomial logistic)**: $p(y=k|x) = e^{w_k^\top x}/\sum_j e^{w_j^\top x}$.

**SVM**: ищет максимально разделяющую гиперплоскость (max margin); в нелинейном варианте — через kernel trick.

**Деревья решений**: рекурсивно делят пространство по признакам, оптимизируя критерий (Gini, entropy).

**Ансамбли**:
- **Random Forest** — bagging решающих деревьев на бутстрапных выборках.
- **Бустинг** (AdaBoost, GBDT, XGBoost, LightGBM, CatBoost) — последовательное добавление слабых учеников, фокусируясь на ошибках.

**KNN**: $f(x) = $ мажоритарный класс среди $k$ ближайших в обучающей выборке. Параметр-свободный, чувствителен к масштабу.

**Naive Bayes**: $p(y|x)\propto p(y)\prod_j p(x_j|y)$ при допущении условной независимости признаков. Быстро, хорошо работает на текстах.

**Нейронные сети**: MLP, CNN (для изображений), RNN/Transformer (для последовательностей).

### Регуляризация и борьба с переобучением
- L1/L2 на коэффициентах модели (Lasso/Ridge).
- Dropout, batch norm, weight decay в нейросетях.
- Early stopping по валидационной кривой.
- Прореживание (pruning) деревьев.

### Несбалансированные классы
- Взвешивание классов в лоссе.
- Oversampling меньшинства (SMOTE) или undersampling большинства.
- Ансамбли с балансировкой.
- Threshold tuning по нужному классу.

### Метрики оценки
Для бинарной: accuracy, precision, recall, F1, ROC-AUC, PR-AUC. Для мультикласса: macro/micro/weighted F1. Подробнее в вопросе 15.

### Пример: логистическая регрессия для спам-фильтра
- $x$ — признаки письма (TF-IDF, длина, домен отправителя…).
- $y\in\{0, 1\}$ — спам / не спам.
- Модель: $p = \sigma(w^\top x)$.
- Обучение: минимизация $-\sum y_i\ln p_i + (1-y_i)\ln(1-p_i)$ + L2-регуляризация.
- Решение: $\hat w$ через L-BFGS.
- Метрика: F1 на validation (важнее accuracy при дисбалансе).

### Подводные камни
- **Утечка**: признаки, недоступные в момент прогноза, или просачивающаяся метка.
- При сильном дисбалансе accuracy обманчива — нужен F1, AUC.
- Калибровка вероятностей (особенно для XGBoost, SVM) — Platt scaling, isotonic regression.
- Несоответствие распределений train/test (covariate shift).
