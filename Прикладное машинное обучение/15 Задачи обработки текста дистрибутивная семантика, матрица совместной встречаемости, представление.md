# Задачи обработки текста: дистрибутивная семантика, матрица совместной встречаемости, представление слов в виде векторов малой размерности. Модель Word2vec.

## TL;DR
**Дистрибутивная семантика** (Harris): «слова с похожим контекстом имеют похожее значение». Базовая реализация — **матрица совместной встречаемости** $M\in\mathbb R^{V\times V}$: $M_{ij}$ = частота слов $i, j$ в окне. Часто перевзвешивают через PPMI; затем понижают размерность (SVD/LSA → плотные эмбеддинги). **Word2vec** (Миколов, 2013) — нейросетевой метод: модели **CBOW** (по контексту предсказать слово) и **Skip-Gram** (по слову — контекст). Обучаются с **Negative Sampling** или иерархическим softmax. Получаются плотные векторы (50–300 d) с замечательным свойством линейности: «king − man + woman ≈ queen». Расширения: GloVe, FastText, контекстные эмбеддинги (ELMo, BERT).

## Развёрнуто

### Дистрибутивная гипотеза
«Words that occur in similar contexts tend to have similar meanings» (Harris, 1954). Семантика слова кодируется через распределение его контекстов в корпусе. Это даёт операциональный способ построить векторные представления, не используя ручную разметку семантики.

### Матрица совместной встречаемости (co-occurrence matrix)
Для словаря $V$ слов и корпуса:
$$M_{ij} = \#\{(\text{позиция}): \text{слово } w_i\text{ и } w_j\text{ встречаются в окне}\}.$$
Окно — $\pm k$ слов вокруг центра.

**Перевзвешивание**:
- **PMI (Pointwise Mutual Information)**:
$$\text{PMI}(i, j) = \log\frac{P(i, j)}{P(i)P(j)}.$$
- **PPMI (Positive PMI)**: $\max(\text{PMI}, 0)$ — отрицательные значения часто шумные, обрезают.

**Понижение размерности (LSA / Truncated SVD)**: $M\approx U\Sigma V^\top$ с топ-$d$ сингулярными значениями. Каждое слово — строка $U_d$ в $\mathbb R^d$, $d = 50–500$. Получаются плотные эмбеддинги.

Преимущества co-occurrence + SVD: математически прозрачно, тренируется одним проходом по корпусу. Недостатки: большая разреженная матрица, дорого для огромных корпусов.

### Word2vec
Нейросетевой подход к векторным представлениям. Две архитектуры:

**CBOW (Continuous Bag-of-Words)**: предсказать центральное слово $w_t$ по контексту $w_{t-k},\dots,w_{t+k}$.
$$\hat v_{\text{ctx}} = \frac{1}{2k}\sum_{j\ne 0} v_{w_{t+j}},\quad P(w_t|\text{ctx}) = \frac{e^{u_{w_t}^\top \hat v_{\text{ctx}}}}{\sum_w e^{u_w^\top \hat v_{\text{ctx}}}}.$$

**Skip-Gram**: по слову $w_t$ предсказать контекст $w_{t+j}$.
$$P(w_{t+j}|w_t) = \frac{e^{u_{w_{t+j}}^\top v_{w_t}}}{\sum_w e^{u_w^\top v_{w_t}}}.$$

Обучается максимизация правдоподобия через SGD. Полные softmax дороги — $O(V)$; ускорения:

**Negative Sampling (SGNS)**: вместо softmax максимизируем
$$\log\sigma(u_o^\top v_c) + \sum_{i=1}^k \mathbb E_{w_i\sim P_n}[\log\sigma(-u_{w_i}^\top v_c)].$$
$P_n$ — шумовое распределение (обычно $\propto f(w)^{3/4}$). $k = 5–20$.

**Hierarchical Softmax**: бинарное дерево слов (Huffman) — сложность $O(\log V)$.

После обучения каждое слово имеет два вектора $v_w, u_w$; на практике используют $v_w$ или $(v_w + u_w)/2$.

### Свойства Word2vec эмбеддингов
- Плотные векторы $\mathbb R^{300}$.
- **Косинусная близость** $\cos(v_a, v_b)$ отражает семантическую близость.
- **Линейные регулярности**: $v_{\text{king}} - v_{\text{man}} + v_{\text{woman}}\approx v_{\text{queen}}$. Аналогично страны/столицы, мн. число и т.п.
- Кластеризация по темам: семантически близкие слова собираются.

### Связь Word2vec и матрицы со-встречаемости
Levy & Goldberg (2014) показали: SGNS неявно факторизует **сдвинутую** PPMI-матрицу: $u_w^\top v_c\approx\text{PMI}(w,c) - \log k$. То есть нейросетевые и матричные методы — две стороны одной медали.

### Расширения

**GloVe** (Global Vectors, Pennington et al., 2014): прямая факторизация co-occurrence matrix:
$$\sum_{i,j} f(M_{ij})(v_i^\top u_j + b_i + b_j - \log M_{ij})^2\to\min,$$
$f$ — функция веса, гасящая редкие пары.

**FastText** (Bojanowski et al., 2017): эмбеддинг слова — сумма эмбеддингов n-грамм символов. Решает OOV (out-of-vocabulary) и работает на морфологически богатых языках.

**ELMo, BERT, GPT**: **контекстные** эмбеддинги — представление слова зависит от его контекста. Базируются на двунаправленных LSTM (ELMo) или Transformer (BERT/GPT). Заменили статические Word2vec в большинстве NLP-задач.

### Применение
- Семантическая близость и поиск.
- Признаки для классификации (sentiment, topic).
- Машинный перевод (входной слой).
- Рекомендации (item2vec, doc2vec).
- Биоинформатика (последовательности белков).

### Пример
```python
from gensim.models import Word2Vec
sentences = [["машинное", "обучение", "круто"], ["глубокое", "обучение", "ещё", "круче"]]
model = Word2Vec(sentences, vector_size=100, window=5, min_count=1, sg=1, negative=5)
model.wv.most_similar("обучение")
```

### Подводные камни
- Word2vec не учитывает многозначность слова (одно слово = один вектор) — решается контекстными моделями.
- Эмбеддинги наследуют **гендерные/расовые предвзятости** корпуса (bias) — есть методы дебайаса, но проблема нерешённая.
- Нужно много данных и аккуратной предобработки (токенизация, нижний регистр, удаление редких слов).
- Косинусная близость отражает «дистрибутивную близость», что не совпадает с лингвистической синонимией (антонимы тоже могут быть близки).
