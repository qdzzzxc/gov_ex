# Задача выявления сообществ в сетях, понятие модулярности, алгоритмы Гирвана-Ньюмана, Лувенский алгоритм, дискретный метод распространения меток.

## TL;DR
**Задача обнаружения сообществ (community detection)** — разбиение узлов сети на группы с большой плотностью внутренних связей и малой между группами. **Модулярность Ньюмана $Q$** — мера качества разбиения: $Q = \frac{1}{2m}\sum_{ij}(A_{ij} - k_i k_j/(2m))\delta(c_i, c_j)$, $\in[-1, 1]$. **Гирван–Ньюман** итеративно удаляет рёбра с максимальной edge-betweenness, строит дендрограмму, выбирает разбиение по максимуму $Q$. **Лувенский алгоритм** — жадная локальная оптимизация $Q$ с многоуровневым агрегированием, $O(n\log n)$, лидер по скорости. **Label Propagation (LPA)** — каждый узел перенимает наиболее частую метку соседей, очень быстрый, $O(m)$, но недетерминированный.

## Развёрнуто

### Задача
Дан граф $G = (V, E)$. Разбить $V$ на непересекающиеся **сообщества** $C_1,\dots,C_K$ так, чтобы рёбер **внутри** сообществ было больше, чем ожидаемо в случайной сети, и меньше **между**. Число $K$ — обычно неизвестно.

Применения: соцсети, биология (модули белковых сетей), рекомендации, обнаружение мошенничества, организация цитирований.

### Модулярность Ньюмана
$$Q = \frac{1}{2m}\sum_{i,j}\left(A_{ij} - \frac{k_i k_j}{2m}\right)\delta(c_i, c_j),$$
$A_{ij}$ — матрица смежности, $k_i$ — степень, $m = |E|$, $c_i$ — сообщество узла $i$, $\delta$ — Кронекер. Эквивалентно:
$$Q = \sum_{c}\left[\frac{e_c}{m} - \left(\frac{d_c}{2m}\right)^2\right],$$
$e_c$ — число рёбер внутри сообщества $c$, $d_c$ — суммарная степень узлов в $c$.

$Q$ сравнивает наблюдаемое с ожидаемым в случайной сети с теми же степенями (configuration model). $Q > 0$ — структура сообществ выраженная; $Q\approx 0{,}3-0{,}7$ для реальных сетей.

**Ограничения**: проблема разрешения (resolution limit) — $Q$ не различает мелкие сообщества ниже определённого размера.

### Алгоритм Гирвана–Ньюмана (2002)
**Идея**: рёбра между сообществами имеют высокую **edge-betweenness centrality** — число кратчайших путей, проходящих через ребро.

Алгоритм:
1. Считаем edge-betweenness всех рёбер.
2. Удаляем ребро с максимальной betweenness.
3. Если граф разбился на компоненты — фиксируем разбиение.
4. Пересчитываем edge-betweenness, повторяем.
5. Получаем **дендрограмму** разбиений; выбираем разрез с максимальным $Q$.

**Сложность**: $O(m^2 n)$ для невзвешенных, $O(m n^2)$ для общих — медленно для больших сетей. Хорош как эталон на малых.

### Лувенский алгоритм (Blondel et al., 2008)
Жадная многоуровневая оптимизация модулярности.

**Фаза 1 (локальное движение)**:
- каждый узел — отдельное сообщество;
- для каждого узла $i$ перебираем перенос в сообщество соседа, выбираем максимальный прирост $\Delta Q$;
- повторяем по всем узлам, пока есть улучшения.

**Фаза 2 (агрегация)**:
- сжимаем каждое сообщество в один узел; рёбра становятся взвешенными суммами рёбер; петли соответствуют внутренним рёбрам;
- возвращаемся к Фазе 1 на новом графе.

Повторяем до отсутствия улучшений. Сложность практически $O(n\log n)$. Стандарт индустрии для больших сетей.

**Прирост модулярности при переносе**: $\Delta Q$ можно вычислить за $O(\deg(i))$ через сохранённые статистики $\sum_{in}, \sum_{tot}$ сообществ.

### Леденовский алгоритм (Leiden, 2019)
Улучшение Лувенского: гарантирует, что сообщества **связны**, и быстрее сходится. На практике замещает Лувенский в больших задачах.

### Label Propagation Algorithm (LPA, Raghavan et al., 2007)
Простой синхронный/асинхронный алгоритм:
1. Каждый узел получает уникальную метку.
2. На каждой итерации каждый узел принимает наиболее частую метку среди соседей (ничьи разрешаются случайно).
3. Повторяем до стабилизации.

**Свойства**:
- Сложность $O(m)$ за итерацию, обычно сходится за $O(\log n)$ — суммарно почти линейно.
- Не требует параметров, не оптимизирует целевую функцию явно.
- Результат **недетерминирован** (зависит от порядка), может давать разные разбиения на разных запусках.
- Часто страдает от тривиального решения (одна большая компонента).

### Другие методы
- **Spectral clustering** — собственные векторы лапласиана $L = D - A$ или нормализованного. Особенно полезно для $K$ известного и для разрезов по нормализованной мере.
- **Infomap** — оптимизация описания случайных блужданий (карта потоков информации).
- **Stochastic block models (SBM)** — байесовский подход, оценивает вероятность связи внутри/между сообществами.
- **Перекрывающиеся** сообщества — Clique Percolation, BigCLAM, ego-networks.

### Сравнительная таблица
| Алгоритм | Сложность | Разбиение | Преимущества | Недостатки |
|---|---|---|---|---|
| Гирван–Ньюман | $O(m^2 n)$ | дендрограмма | интерпретируем | очень медленно |
| Лувенский | $O(n\log n)$ | плоское | быстрый, точный | resolution limit |
| Leiden | $O(n\log n)$ | плоское | гарантия связности | посложнее |
| LPA | $O(m)$ | плоское | очень быстрый | недетерминирован |
| Spectral | $O(n^3)$ для прямого | плоское | теоретически обоснован | плохо масштабируется |

### Пример
Karate Club Захари (стандартный бенчмарк, 34 узла): Гирван–Ньюман даёт 2 сообщества в соответствии с реальным расколом клуба, $Q\approx 0{,}40$. Лувенский — 4 более мелких сообщества с $Q\approx 0{,}45$ (из-за resolution limit).

### Подводные камни
- Модулярность не всегда отражает «истинную» структуру; разные алгоритмы дают разные ответы.
- Resolution limit Лувенского — мелкие сообщества склеиваются.
- В разреженных или иерархических сетях нужно осторожно интерпретировать результат.
- Сравнение с эталонной разметкой — через NMI, ARI, F-меру.
