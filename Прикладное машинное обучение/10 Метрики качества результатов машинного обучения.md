# Метрики качества результатов машинного обучения. Метрики качества моделей регрессии и классификации. MAE, MSE, Precision, Recall, F1, ROC-AUC. Понятие недо- и переобучения. Диагностика систем машинного обучения. Использование кривых обучения.

### Метрики регрессии
- **MAE** = $\frac{1}{n}\sum|y_i - \hat{y}_i|$ — робастна к выбросам, в единицах $y$.
- **MSE** = $\frac{1}{n}\sum(y_i - \hat{y}_i)^2$ — сильнее штрафует выбросы.
- **RMSE** = $\sqrt{MSE}$ — в единицах $y$, удобно интерпретировать.
- **$R^2$** = $1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}$ — доля объяснённой дисперсии. 1 идеал, 0 не лучше среднего, $<0$ хуже среднего.
- **MAPE** = средняя процентная ошибка. Удобен для бизнеса, плох при $y \to 0$.

### Confusion matrix и базовые метрики классификации

**TP, FP, TN, FN** — все метрики через них.
- **Accuracy** = $(TP+TN)/n$. Обманчива при дисбалансе.
- **Precision** = $TP/(TP+FP)$. Из предсказанных позитивов — сколько реальных.
- **Recall (TPR)** = $TP/(TP+FN)$. Из реальных позитивов — сколько поймали.
- **F1** = $2PR/(P+R)$ — гармоническое среднее. $F_\beta$ для асимметричной важности.

**Когда что важнее:** recall важен в медицине/безопасности (пропуск катастрофа), precision важен в спам-фильтрах/рекомендациях (ложная тревога дорогая).

### ROC-кривая и AUC

**ROC** — график **TPR vs FPR** при разных порогах. FPR = $FP/(FP+TN)$ — доля негативов, ошибочно названных позитивами.

Граничные точки: $(0,0)$ при высоком пороге, $(1,1)$ при низком. Идеал — в (0,1). Случайный классификатор — диагональ.

**AUC-ROC** = площадь под ROC, $\in [0, 1]$.
- **Интерпретация:** $P(g(x^+) > g(x^-))$ — вероятность того, что случайный позитив получит больший скор, чем случайный негатив.
- AUC = 1 идеал, 0.5 случай, 0 стабильно неправильно (инвертируем).

**Преимущества:**
- Инвариантна к порогу — оценка модели целиком.
- Не зависит от калибровки скоров.
- Базовый уровень всегда 0.5, сравнима между датасетами.

**Минус ROC при сильном дисбалансе:** TN огромное → FPR маленький даже при многих FP → оптимистичная картина. Использовать PR-AUC.

### PR-кривая и AUC-PR

**PR-кривая** — график **Precision vs Recall** при разных порогах. Не использует TN.

**Когда брать PR-AUC:**
- Сильный дисбаланс (1% позитивов и меньше).
- Информационный поиск, fraud detection, рекомендации.

**Минусы PR-AUC:**
- Базовый уровень = доля позитивов, зависит от дисбаланса.
- Не сравнима между датасетами с разным соотношением классов.
- Нет красивой вероятностной интерпретации.
- Менее гладкая, более шумная на маленьких тестах.

**Правило:** при сбалансированных классах — ROC, при сильном дисбалансе — PR. На практике часто смотрят обе.

### Log-loss

$-\frac{1}{n}\sum [y_i \ln p_i + (1-y_i)\ln(1-p_i)]$ — учитывает вероятностные предсказания, сильно штрафует уверенно неправильные.

### Мультикласс
- **Macro F1** — среднее по классам без весов (равенство классов).
- **Micro F1** — суммируем TP/FP/FN по классам, потом считаем F1.
- **Weighted F1** — взвешено по поддержке.
- **Top-k accuracy** — правильный ответ среди $k$ предложенных.

### Недо- и переобучение

**Underfitting (high bias):** модель слишком простая.
- Ошибка train **высокая**, val **близко к train**.
- Решения: усложнить модель, добавить признаки, уменьшить регуляризацию.

**Overfitting (high variance):** модель запомнила обучение.
- Train **низкая**, val **заметно выше**, **большой gap**.
- Решения: больше данных, регуляризация (L1/L2, dropout), упрощение, ансамбли, early stopping.

### Bias-variance trade-off

$$\mathbb{E}[(y - \hat{y})^2] = \text{Bias}^2 + \text{Variance} + \sigma^2_{\text{noise}}.$$

- Простая модель: bias↑, variance↓ (underfitting). Логрег, линейная.
- Сложная: bias↓, variance↑ (overfitting). Глубокое дерево, нейросеть.
- Идеал — баланс.

**Double descent:** в нейросетях после порога интерполяции test ошибка может снова падать.

### Кривые обучения

**Learning curves** — графики ошибок train и val в зависимости от:
- **размера выборки** $n$:
    - underfitting: обе кривые сходятся высоко, gap маленький.
    - overfitting: train низко, val выше, gap большой; больше данных закрывает gap.
- **сложности модели**:
    - train ошибка монотонно падает.
    - val ошибка U-образна: минимум при оптимальной сложности.

**Validation curve** — train/val в зависимости от гиперпараметра.
### Диагностика модели
1. train + val оба высокие → underfitting → усложняй.
2. train низкая + val высокая → overfitting → регуляризуй.
3. Обе одинаковые, не растут с $n$ → ограничение модели.
4. Расхождение train/val уменьшается с $n$ → нужно больше данных.
### Кросс-валидация
- **$k$-fold:** $k$ разбиений, обучаем $k$ раз, усредняем метрику.
- **Stratified $k$-fold:** сохраняет пропорции классов.
- **Time-series CV:** без перемешивания, forward chaining.
### Подводные камни
- Метрика на test ≠ оптимизируемый лосс.
- Test-set leakage — переиспользование теста как валидации → оптимистичная оценка.
- При дисбалансе accuracy 99% бессмысленна — нужны recall, F1, AUC.
- Калибровка вероятностей: высокая AUC не значит хорошо калиброванные вероятности (нужны Platt scaling, isotonic regression).
### Главное на экзамене
1. **Регрессия:** MAE/MSE/RMSE/R² — формулы и разница.
2. **Классификация:** precision/recall — формулы, F1 как их комбинация.
3. **ROC-AUC** — TPR vs FPR, вероятностная интерпретация, инвариантность к порогу.
4. **PR-AUC** — при сильном дисбалансе.
5. **Underfitting vs overfitting** — признаки и решения.
6. **Bias-variance trade-off** — формула декомпозиции.
7. **Кривые обучения** — что показывают, как диагностируют.
