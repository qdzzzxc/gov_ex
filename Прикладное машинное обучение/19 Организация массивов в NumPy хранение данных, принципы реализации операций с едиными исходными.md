# Организация массивов в NumPy: хранение данных, принципы реализации операций с едиными исходными данными. Универсальные функции и применение функций по осям в NumPy, маскирование и прихотливое индексирование в NumPy.

## TL;DR
**NumPy ndarray** — однородный многомерный массив фиксированного `dtype`, хранится как **непрерывный** (или strided) блок памяти, с метаданными `shape`, `strides`, `dtype`. Операции — **векторизованные** через **универсальные функции (ufuncs)**: одна C-функция применяется поэлементно к массивам, поддерживая **broadcasting** (расширение размерностей по правилам). Применение по осям — `np.sum(a, axis=0)`, `apply_along_axis`, `reduce/accumulate` ufunc'ов. **Маскирование** — `a[a>0]` (булева индексация). **Прихотливая индексация (fancy)** — `a[[0,2,5]]`, `a[[0,1],[2,3]]` — индексы массивами целых.

## Развёрнуто

### Хранение данных
`ndarray` — обёртка над непрерывным буфером памяти + метаданные:
- `dtype` — тип элементов (`int32`, `float64`, `bool`, structured dtype для записей).
- `shape` — кортеж размерностей $(d_0, d_1, \dots)$.
- `strides` — байтовые шаги по каждой оси: чтобы попасть в `a[i, j]`, отступ от базы `i*strides[0] + j*strides[1]`.
- `order`: 'C' (row-major, как в C) или 'F' (column-major, как в Fortran).

**Strided memory** позволяет:
- срезам (slice) `a[1:4]` не копировать данные, а лишь менять `shape/strides`;
- транспонированию `a.T` быть бесплатным (просто меняет `strides`);
- broadcasting работать через нулевые `strides` по «недостающим» осям.

`a.flags['C_CONTIGUOUS']` показывает непрерывность.

### Операции с едиными исходными данными
Главный принцип: **одну операцию** применяем к **большому** массиву одной командой, а не цикл в Python. Это **векторизация** — внутри NumPy всё уходит в SIMD/BLAS C-код.

**Universal functions (ufuncs)** — поэлементные операции, реализованные на C:
- арифметика `+, -, *, /, **, %`;
- математика `np.sin, np.exp, np.log, np.sqrt, np.maximum, np.where`;
- сравнения `==, <, >`;
- логика `np.logical_and/or/not`.

Каждая ufunc:
- **vectorized** — применяется к массиву поэлементно;
- поддерживает **broadcasting** — несовпадающие размерности расширяются;
- имеет методы `reduce`, `accumulate`, `outer`, `reduceat`.

**Broadcasting**: правило выравнивания форм с конца:
- размерности должны совпадать **или** одна из них должна быть 1.
- Размерности 1 «растягиваются» (виртуально) до нужного размера.

Пример: `a.shape = (3, 1)`, `b.shape = (1, 4)` → результат `(3, 4)`.

### Применение функций по осям
Параметр `axis` у агрегатов:
```python
a = np.arange(12).reshape(3, 4)
a.sum()           # 66 — по всему массиву
a.sum(axis=0)     # [12, 15, 18, 21] — по столбцам (свёртка по строкам)
a.sum(axis=1)     # [6, 22, 38] — по строкам
a.mean(axis=0)
a.std(axis=1, keepdims=True)  # сохранение размерности
```
`axis` — какая ось «сворачивается» (исчезает). `keepdims=True` оставляет ось размером 1 — удобно для последующего broadcasting.

**`np.apply_along_axis(func, axis, arr)`** — применить произвольную Python-функцию к каждому 1D-срезу вдоль `axis`. Удобно, но медленно (Python-цикл внутри).

**ufunc-методы**:
- `np.add.reduce(a, axis=0)` — то же, что `a.sum(axis=0)`;
- `np.add.accumulate(a)` — кумулятивная сумма (`cumsum`);
- `np.multiply.outer(a, b)` — внешнее произведение по любой ufunc.

### Способы индексирования

**1. Базовая индексация / срезы**: `a[1, 2]`, `a[1:4, ::2]`. Возвращает **view** (общую память) — изменение влияет на исходный массив.

**2. Булева индексация (маскирование)**:
```python
a = np.array([3, -1, 4, -2, 5])
mask = a > 0       # array([True, False, True, False, True])
a[mask]            # array([3, 4, 5]) — копия
a[a < 0] = 0       # in-place замена
```
Полезно для фильтрации и условной модификации.

**3. Прихотливая (fancy) индексация — массивы целых**:
```python
a = np.arange(10)
a[[0, 2, 5]]       # array([0, 2, 5]) — отбор по индексам
a = np.arange(12).reshape(3, 4)
a[[0, 2], [1, 3]]  # array([1, 11]) — пары (0,1), (2,3)
a[:, [3, 0, 1]]    # переставление столбцов
```
Возвращает **копию**.

**4. Комбинированная**: `a[1:, [0, 2]]`, `a[mask, :]`.

**5. `np.where(cond, x, y)`** — выбор элементов из двух массивов по условию: $(\text{cond}_i\,?\,x_i\,:\,y_i)$.

### Универсальные функции — расширения
- **Custom ufunc**: `np.frompyfunc(f, nargs, nouts)` или Numba `@vectorize`.
- **Vectorize**: `np.vectorize(f)` — обёртка вокруг Python-функции (внутри цикл, не быстрее, но удобно).
- **`out=` и `where=`**: запись результата в существующий массив, условное применение.

### Производительность
Векторизация даёт ускорение в 10–100x по сравнению с Python-циклом. Дополнительно:
- **BLAS/LAPACK** для линейной алгебры (`np.linalg.solve`, `dot`, `matmul`);
- **broadcasting** избегает копий;
- **в-place** операции `+=`, `*=` не выделяют новую память;
- **`einsum`** — обобщённое тензорное сжатие.

### Пример: нормализация по строкам
```python
import numpy as np
A = np.random.rand(1000, 50)
mean = A.mean(axis=1, keepdims=True)
std  = A.std(axis=1, keepdims=True)
A_norm = (A - mean) / std
```
Один проход без явного цикла, broadcasting по оси 1.

### Маскирование с `np.ma`
`np.ma.masked_array` — массив с маской, где помеченные значения исключаются из агрегатов:
```python
ma = np.ma.array([1, 2, 3, 4], mask=[0, 1, 0, 1])
ma.mean()  # 2.0 — учитывает только 1 и 3
```

### Пример: прихотливое индексирование для перестановки
```python
a = np.array(['a', 'b', 'c', 'd', 'e'])
order = np.array([3, 0, 4, 1, 2])
a[order]  # array(['d', 'a', 'e', 'b', 'c'])
```

### Подводные камни
- **View vs copy**: базовые срезы — view, fancy/булева — copy. Не путать.
- **Изменение dtype**: `a.astype(...)` создаёт копию; неявные преобразования могут быть дорогими.
- **Неконтролируемый broadcasting** иногда даёт неожиданные результаты — стоит явно проверять `shape`.
- **Слишком большие массивы**: NumPy всё держит в памяти; для больших данных — Dask Array, Zarr.
- Универсальные функции через `np.vectorize`/`frompyfunc` **не быстрее** Python-цикла; ускорение — в реальной C/SIMD реализации (Numba, Cython, native NumPy).
