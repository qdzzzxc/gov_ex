# Организация массивов в NumPy: хранение данных, принципы реализации операций с едиными исходными данными. Универсальные функции и применение функций по осям в NumPy, маскирование и прихотливое индексирование в NumPy.

## Хранение ndarray
Массив — **непрерывный блок памяти** + метаданные:
- **`dtype`** — однородный тип элементов (`int32`, `float64`). Однородность — главное отличие от Python-списка.
- **`shape`** — кортеж размерностей $(d_0, d_1, \dots)$.
- **`strides`** — байтовые шаги по каждой оси. Чтобы попасть в `a[i, j]`: $\text{offset} = i \cdot \text{strides}[0] + j \cdot \text{strides}[1]$.
- **`order`** — `'C'` (row-major) или `'F'` (column-major). Strided memory делает бесплатными:
- Срезы `a[1:4]` — не копируют, меняют `shape/strides`.
- Транспонирование `a.T` — меняет порядок `strides`, память не трогается.
- Broadcasting — нулевые `strides` по виртуальной оси.
## Векторизация и ufuncs
**Универсальные функции (ufuncs)** — поэлементные операции, реализованные на C: `+, -, *, /, np.sin, np.exp, np.maximum`. Почему быстрее Python-цикла в 10–100 раз:
1. C-код вместо интерпретатора.
2. SIMD (AVX/SSE) — одна инструкция на 4–16 значений.
3. Локальность памяти (непрерывный буфер vs массив указателей). Не из-за мультипроцессинга — NumPy однопоточный. Параллелизм только в BLAS (`dot`, `matmul`). Методы ufunc: `np.add.reduce`, `np.add.accumulate`, `np.multiply.outer`.
## Broadcasting
Правило выравнивания форм **с конца**:
- Размерности совпадают или одна равна 1.
- Размерность 1 виртуально растягивается. Примеры:
- `(3, 4) + (4,)` → `(3, 4) + (1, 4)` → `(3, 4)`.
- `(3, 1) + (1, 4)` → `(3, 4)`.
- `(3, 4) + (3,)` → ОШИБКА. Позволяет избежать явного дублирования маленьких массивов.
## Применение по осям
`axis` — какая ось **сворачивается** (исчезает):
```python
a.shape == (3, 4)
a.sum(axis=0)  # форма (4,) — свернули по строкам, остались столбцы
a.sum(axis=1)  # форма (3,) — свернули по столбцам, остались строки
```
`keepdims=True` оставляет ось размером 1 — удобно для последующего broadcasting:
```python
mean = A.mean(axis=1, keepdims=True)  # (1000, 1) вместо (1000,)
A_norm = (A - mean) / std              # broadcasting работает
```
`np.apply_along_axis(func, axis, arr)` — произвольная Python-функция на 1D-срезы. Медленно (внутри Python-цикл).
## Три типа индексации
**1. Базовая (срезы):** целые и slice, например `a[1, 2]`, `a[1:4, ::2]`. Возвращает **view** — общая память. **2. Булева (маскирование):**
```python
a[a > 0]         # элементы по условию
a[a < 0] = 0     # in-place условная замена
```
Возвращает **copy**. **3. Прихотливая (fancy):** массивы целых индексов:
```python
a[[0, 2, 5]]              # 3 элемента по индексам
A[[0, 2], [1, 3]]         # пары (0,1) и (2,3)
A[:, [3, 0, 1]]           # перестановка столбцов
```
Возвращает **copy**. Можно комбинировать: `a[1:, [0, 2]]`, `a[mask, :]`. Поэлементный тернарный — `np.where(cond, x, y)`.
## View vs copy

| Операция                                   | Результат            |
| ------------------------------------------ | -------------------- |
| Базовая индексация, срезы                  | view                 |
| `a.T`, `reshape`                           | view (если возможно) |
| Булева индексация                          | copy                 |
| Прихотливая индексация                     | copy                 |
| `a.astype()`, `a.copy()`                   | copy                 |
| Проверка: `a.base is not None` → это view. |                      |
