# Рекуррентные нейронные сети, принципы их обучения. Сложности применения рекуррентных нейронных сетей. Модуль LSTM.

## Зачем нужны RNN
Обработка последовательностей переменной длины (текст, временные ряды, логи). FC требует фиксированный размер входа и не использует временную структуру.
## Vanilla RNN

$$h_t = \tanh(W_h h_{t-1} + W_x x_t + b), \quad y_t = W_y h_t + c$$ Параметры $(W_h, W_x, W_y)$ **разделяются между всеми шагами** — те же веса применяются на каждом $t$. Отсюда «рекуррентная». Применения: классификация последовательности, seq2seq (перевод), tagging по шагам.
## Обучение: BPTT (Backpropagation Through Time)
Сеть разворачивается по времени в глубокую сеть на $T$ шагов с разделёнными весами, дальше обычный backprop. Градиент по $W_h$ содержит произведение $\prod_s \partial h_s / \partial h_{s-1}$ — отсюда все проблемы. **Truncated BPTT:** разворачиваем только последние $k$ шагов, экономия памяти ценой потери очень длинных зависимостей.
## Главные проблемы
1. **Vanishing gradient** — произведение $\partial h_s/\partial h_{s-1} = W_h^\top \mathrm{diag}(\sigma'(\cdot))$ при $|W_h| < 1$ экспоненциально стремится к нулю → сеть не учит длинные зависимости.
2. **Exploding gradient** — при $|W_h| > 1$ экспоненциально растёт. Решение: **gradient clipping** $g \leftarrow g \cdot \min(1, \tau/|g|)$.
3. **Ограниченная память** — на практике vanilla RNN запоминает 10–20 шагов.
4. **Невозможность параллелизации** по $t$ — шаги последовательны, медленно на GPU. Поэтому в современном NLP RNN вытеснены Transformer'ами.
## LSTM
Добавляет **клеточное состояние** $c_t$ — отдельную «магистраль памяти» + три гейта (sigmoid → $[0, 1]$):
- **forget** $f_t = \sigma(W_f [h_{t-1}, x_t] + b_f)$ — что забыть из $c_{t-1}$.
- **input** $i_t = \sigma(W_i [h_{t-1}, x_t] + b_i)$ — что записать.
- **output** $o_t = \sigma(W_o [h_{t-1}, x_t] + b_o)$ — что выдать наружу. **Кандидат на запись:** $\tilde{c}_t = \tanh(W_c [h_{t-1}, x_t] + b_c)$. **Обновление:** $$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t, \quad h_t = o_t \odot \tanh(c_t)$$ **Почему работает:** путь $c_{t-1} \to c_t$ **аддитивный**, $\partial c_t/\partial c_{t-1} \approx f_t$. Если $f_t \approx 1$ — градиент не затухает экспоненциально, длинные зависимости обучаются. Это и решает vanishing gradient. Существует упрощённая версия — **GRU** с двумя гейтами вместо трёх и без отдельного $c_t$. На практике сравнима с LSTM.
