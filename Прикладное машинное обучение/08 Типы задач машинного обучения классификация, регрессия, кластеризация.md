# Типы задач машинного обучения: классификация, регрессия, кластеризация. Обучение с учителем и без учителя. Машинное обучение как оптимизационная задача.

### Парадигмы
**Обучение с учителем (supervised):** есть размеченные пары $(x, y)$. Цель — найти $f: \mathcal{X} \to \mathcal{Y}$, обобщающуюся на новые данные.

- **Классификация** — дискретный $y$: бинарная, мультикласс, мультилейбл. Лоссы: cross-entropy, hinge.
- **Регрессия** — непрерывный $y \in \mathbb{R}$. Лоссы: MSE, MAE, Huber.

**Обучение без учителя (unsupervised):** только ${x_i}$. Задачи:
- **Кластеризация** — разбить на группы (k-means, DBSCAN, иерархическая, GMM, спектральная).
- **Понижение размерности** — выжать суть (PCA линейное; t-SNE, UMAP, автокодировщики нелинейные).
- **Оценка плотности** — выучить $P(x)$ (KDE, GMM, normalizing flows).
- **Поиск аномалий** — выделить выбросы.
- **Поиск ассоциаций** — Apriori, FP-growth.

**Другие парадигмы:**

- **RL** — агент, среда, награды, политика. Q-learning, policy gradient.
- **Semi-supervised** — мало размеченных + много без меток.
- **Self-supervised** — искусственные метки из самих данных (Word2vec, BERT, MAE).
- **Active learning** — модель выбирает, что разметить.
- **Transfer learning, meta-learning, few-shot.**

### Зачем понижение размерности

1. Борьба с **проклятием размерности**.
2. **Визуализация** (2D/3D).
3. Шумоподавление.
4. Сжатие.
5. Ускорение последующих алгоритмов.

### ML как оптимизационная задача

**Эмпирический риск** — средний лосс на выборке: $$\hat{R}(\theta) = \frac{1}{n} \sum_{i=1}^{n} L(f_\theta(x_i), y_i).$$

**Обучение модели** = минимизация: $$\hat{\theta} = \arg\min_\theta \hat{R}(\theta) + \lambda \Omega(\theta).$$

Это формальная постановка **любой supervised задачи**.

**Регуляризация $\Omega(\theta)$** — штраф на сложность параметров:

- **L1 (Lasso):** $|\theta|_1$ — занулят неважные веса (feature selection).
- **L2 (Ridge):** $|\theta|_2^2$ — сжимает все веса, уменьшая переобучение.
- **Dropout** — стохастическая регуляризация для нейросетей.

### Эмпирический vs истинный риск

**Истинный риск** $R(\theta) = \mathbb{E}_{(x, y) \sim P} L(f_\theta(x), y)$ — то, что мы хотим минимизировать (ожидание лосса на всём распределении).

**Проблема:** $P$ неизвестно, считаем $\hat R$ на выборке как приближение $R$. Если выборка маленькая или модель сложная — $\hat R$ мал, но $R$ велик → **переобучение**.

Разница $|R - \hat R|$ ограничивается теорией обобщения (VC dimension, Радемахер).

### Способы решения оптимизационной задачи

1. **Аналитический (closed-form)** — линейная регрессия: $\hat \theta = (X^\top X)^{-1} X^\top y$.
2. **Градиентные методы** — SGD, momentum, Adam. Для гладких функций. Стандарт для нейросетей.
3. **Специализированные:**
    - **EM** для GMM и скрытых переменных.
    - **Координатный спуск** для Lasso.
    - **L-BFGS** — квази-Ньютон.
    - **Жадные** для деревьев и бустинга.

### Связь компонентов

|Компонент|Что|
|---|---|
|Лосс $L$|что минимизируем|
|Гипотеза $\mathcal{F}$|какие функции рассматриваем|
|Регуляризация $\Omega$|контроль сложности|
|Алгоритм оптимизации|как находим $\hat \theta$|
|Метрика на тесте|что измеряем (может ≠ лоссу)|

### Лосс ≠ метрика — пример

Обучаем через **cross-entropy**, оцениваем через **AUC-ROC** / **recall** / **F1**. Это нормально: лосс должен быть дифференцируемым (для градиента), метрика — отражать бизнес-задачу.

### Подводные камни

- **Утечка данных** (data leakage) — признаки из будущего или просочившиеся метки.
- **Concept drift** — распределение в проде ≠ обучающее.
- **Лосс ≠ метрика** — оптимизировать одно, оценивать другое.
- Unsupervised — нет ground truth, валидация субъективна.

### Главное на экзамене

1. **Supervised vs unsupervised** — наличие меток.
2. **Классификация vs регрессия** — дискретный vs непрерывный $y$.
3. **Формула эмпирического риска** + регуляризация.
4. **Эмпирический vs истинный риск** — концептуальная разница, проблема обобщения.
5. **3 способа решения** — аналитический, градиентный, специализированный.
