# Принцип организации обучения многослойных искусственных нейронных сетей: механизм обратного распространения ошибки дифференцируемое программирование и граф потока вычислений.

## TL;DR
Многослойный перцептрон (MLP) — композиция аффинных слоёв и нелинейностей: $h^{(l)} = \sigma(W^{(l)} h^{(l-1)} + b^{(l)})$. Обучение — минимизация лосса $L(\theta)$ по параметрам $\theta = \{W^{(l)}, b^{(l)}\}$ методом стохастического градиентного спуска. **Обратное распространение ошибки (backprop)** — эффективное вычисление градиента через цепное правило: проход вперёд для вычисления активаций и лосса, проход назад для накопления частных производных. **Дифференцируемое программирование** — современный взгляд: модель — это произвольная программа, представленная как **граф потока вычислений (computation graph)**; автоматическое дифференцирование (autograd) даёт градиент любой композиции. PyTorch/TensorFlow/JAX реализуют именно эту идею.

## Развёрнуто

### Многослойная сеть прямого распространения
$$h^{(0)} = x,\quad h^{(l)} = \sigma_l(W^{(l)} h^{(l-1)} + b^{(l)}),\ l=1,\dots,L,\quad \hat y = h^{(L)}.$$
$\sigma_l$ — нелинейная активация: ReLU $\max(0, z)$, sigmoid $1/(1+e^{-z})$, tanh, GELU, softmax (на выходе классификатора).

Функция потерь $L(\hat y, y)$: MSE для регрессии, кросс-энтропия для классификации.

Полная функция: $\theta\mapsto L(f_\theta(x), y)$, где $\theta$ — все веса и смещения.

### Принцип обучения
Эмпирический риск:
$$\hat R(\theta) = \frac{1}{n}\sum_{i=1}^n L(f_\theta(x_i), y_i).$$
**SGD** (мини-партии $B$):
$$\theta_{t+1} = \theta_t - \eta_t\,\nabla_\theta\hat R_B(\theta_t).$$
$\eta_t$ — learning rate. Расширения: momentum, Nesterov, Adam, RMSProp, AdaGrad. Расписания (schedulers): cosine annealing, step decay, warmup.

### Обратное распространение ошибки (backprop)
Алгоритм вычисления $\nabla_\theta L$ за **одну** пару проходов прямой/обратной.

**Прямой проход**: вычисляем $h^{(0)}, z^{(l)} = W^{(l)} h^{(l-1)} + b^{(l)}, h^{(l)} = \sigma_l(z^{(l)})$ до выхода и лосс $L$.

**Обратный проход**: цепное правило. Введём $\delta^{(l)} = \partial L/\partial z^{(l)}$.
- На выходе: $\delta^{(L)} = \nabla_{\hat y} L\,\odot \sigma_L'(z^{(L)})$. Для softmax + кросс-энтропии: $\delta^{(L)} = \hat y - y$.
- Рекурсия: $\delta^{(l)} = (W^{(l+1)\top}\delta^{(l+1)})\odot \sigma_l'(z^{(l)})$.
- Градиенты по параметрам:
$$\frac{\partial L}{\partial W^{(l)}} = \delta^{(l)} (h^{(l-1)})^\top,\quad \frac{\partial L}{\partial b^{(l)}} = \delta^{(l)}.$$

**Сложность**: $O(\text{forward})$ — обратный проход не дороже, чем прямой.

### Дифференцируемое программирование
Современный фреймворк: модель — это **программа**, состоящая из дифференцируемых операций. Каждая операция (сложение, умножение, свёртка, attention, активация) знает, как вычислить свой выход и градиент по своим входам.

**Граф потока вычислений** (computation graph) — DAG, узлы которого — операции и тензоры, рёбра — потоки данных. При прямом проходе строится сам граф; при обратном — обходится в обратном порядке, на каждом узле применяется правило backprop.

**Автоматическое дифференцирование (autograd)** — два режима:
- **forward-mode**: вычисляет $\partial f/\partial x$ за один проход; эффективно при $\dim x$ малом.
- **reverse-mode** (то же, что backprop): вычисляет $\partial f/\partial x$ для скалярного $f$ за один проход; эффективно при $\dim x$ большом — стандарт в DL.

Современные фреймворки:
- **PyTorch**: динамический граф (define-by-run), удобен для отладки.
- **TensorFlow 2 / Keras**: tf.function — компиляция статического графа из питона.
- **JAX**: функциональный autograd через `jax.grad`, JIT-компиляция, vmap, pmap.

Дифференцировать можно произвольные программы: с условиями (через REINFORCE/Gumbel), циклами (распаковка), даже работу с дискретными структурами (через relaxations).

### Особенности оптимизации нейросетей
- **Невыпуклый ландшафт**: множество локальных минимумов, седловых точек, плато. SGD на практике находит хорошие решения.
- **Vanishing/exploding gradients**: для глубоких сетей градиенты либо затухают (sigmoid/tanh), либо взрываются. Решения: ReLU и её варианты, residual connections, weight initialization (He, Xavier), batch normalization.
- **Регуляризация**: L2 (weight decay), dropout, label smoothing, data augmentation, early stopping.
- **Инициализация**: He ($\sqrt{2/n_{in}}$ для ReLU), Xavier ($\sqrt{1/n}$ для tanh).

### Пример: один шаг backprop
2-слойная сеть для бинарной классификации, sigmoid + MSE:
$z = W^{(1)}x + b^{(1)}$, $h = \sigma(z)$, $\hat y = w^{(2)\top}h + b^{(2)}$, $L = (\hat y - y)^2$.

Обратный проход:
$\partial L/\partial \hat y = 2(\hat y - y)$.
$\partial L/\partial w^{(2)} = 2(\hat y - y) h$, $\partial L/\partial b^{(2)} = 2(\hat y - y)$.
$\delta = 2(\hat y - y) w^{(2)}\odot h(1-h)$.
$\partial L/\partial W^{(1)} = \delta x^\top$, $\partial L/\partial b^{(1)} = \delta$.

### Подводные камни
- Производные нелинейностей: ReLU не дифференцируема в 0 (на практике берут субградиент 0 или 1 — не критично).
- Численная нестабильность softmax + log: используется log-sum-exp trick.
- При несбалансированных данных или неправильной инициализации обучение «зависает».
- Не у всех операций есть градиент (argmax, sampling) — нужно репараметризовать.
