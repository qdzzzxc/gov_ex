# Кластеризация как задача машинного обучения. Метод ближайших соседей (kNN) и его обобщения. Подбор числа k. Методы метрической классификации в библиотеке Scikit-Learn.

### Кластеризация — постановка

Обучение без учителя: разбить ${x_i}$ на группы похожих. "Похожесть" — метрика (евклидова, косинусная) или ядро.

### Алгоритмы кластеризации

**k-means.** Минимизация WCSS (inertia): $$J = \sum_{k=1}^K \sum_{x \in C_k} |x - c_k|^2.$$

Алгоритм Ллойда: инициализация центров → назначение точек ближайшему центру → пересчёт центров как средние → повтор. Сложность $O(nKd \cdot \text{iter})$.

Подводные камни: чувствителен к выбросам, любит сферические кластеры, зависит от инициализации (k-means++ помогает).

**Иерархическая** (агломеративная): каждая точка свой кластер → последовательно объединяем два ближайших → дерево вложенных кластеров. Результат — **дендрограмма**, разбиение получается разрезанием на нужной высоте. Метрики связи: single, complete, average, **Ward** (популярный). Дорого: $O(n^3)$.

**DBSCAN.** По плотности: кластер = связное множество точек, у каждой в $\varepsilon$-окрестности есть $\ge$ minPts соседей. Плюсы: не нужно $K$, произвольные формы кластеров, выделяет шум. Параметры: $\varepsilon$, minPts.

**GMM.** Вероятностная: $p(x) = \sum \pi_k \mathcal{N}(x | \mu_k, \Sigma_k)$, обучается EM. Soft-кластеризация.

**Spectral clustering.** Через собственные векторы лапласиана графа сходства + k-means в спектральном пространстве. Хороша для нерегулярных кластеров.

### Подбор числа кластеров $K$

- **Метод локтя:** график $J(K)$, ищем "излом". Субъективно.
- **Силуэт:** $s_i = (b_i - a_i)/\max(a_i, b_i)$, где $a_i$ — среднее внутри своего кластера, $b_i$ — до ближайшего чужого. $s \in [-1, 1]$, выше — лучше. Глобально — среднее.
- **Gap statistic, Calinski-Harabasz, Davies-Bouldin.**
- **BIC** для GMM.

### kNN — метрический классификатор

Для классификации $x$: найти $k$ ближайших обучающих примеров → мажоритарное голосование. Для регрессии — среднее $y$ соседей.

**Lazy:** нет обучения, вся работа на этапе предсказания.

**Метрики:** евклидова, манхэттенская, Минковского, косинусная. **Масштабирование признаков обязательно** (StandardScaler) — иначе одна шкала признаков будет доминировать.

### Подбор $k$

- Малое $k$ → высокая variance (чувствительность к шуму).
- Большое $k$ → высокий bias (сглаживание).
- **Кросс-валидация** по сетке $k = 1, 3, 5, 7, 11, \dots$
- Эмпирика: $k \approx \sqrt{n}$ как старт.
- **Нечётное** $k$ для бинарной — избежать ничьих.

### Обобщения

- **Weighted kNN** — голос соседа взвешен обратно расстоянию (`weights='distance'`).
- **Radius Neighbors** — все соседи в радиусе $r$.
- **Nearest Centroid** — класс ассоциирован с центроидом.
- **LWR (Locally Weighted Regression)** — kNN с локальной моделью.

### Ускорение поиска соседей

- **KD-Tree** — бинарное дерево по осям, $O(\log n)$ в среднем для малых $d$.
- **Ball-Tree** — гиперсферические разбиения, лучше при средних $d$.
- **HNSW, FAISS, Annoy** — приближённые методы для больших $d$ и $n$, используются в продакшене.

sklearn: `algorithm='auto' | 'kd_tree' | 'ball_tree' | 'brute'`.

### Проклятие размерности для kNN

При большом $d$ расстояния между точками **выравниваются** — все точки оказываются примерно равноудалёнными. Близкие соседи становятся неотличимы от далёких → kNN теряет смысл.

**Решение:** перед kNN — PCA, UMAP, feature selection.

Также: большие $n$ + наивный $O(n)$ поиск → тормозит. Использовать индексные структуры или приближённые методы.

### Главное на экзамене

1. **Кластеризация vs классификация** — без меток vs с метками.
2. **k-means** — алгоритм Ллойда, целевая функция WCSS.
3. **DBSCAN** — по плотности, не нужно $K$, выделяет шум.
4. **Силуэт** — формула и интерпретация.
5. **kNN** — голосование соседей; lazy; масштабирование обязательно.
6. **Подбор $k$** — кросс-валидация, $\sqrt n$, нечётное.
7. **Проклятие размерности** — главная проблема kNN при больших $d$.
