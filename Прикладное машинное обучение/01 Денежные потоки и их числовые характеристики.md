### Денежный поток
Последовательность платежей ${C_0, C_1, \dots, C_n}$ в моменты $t_0 = 0, t_1, \dots, t_n$. Положительные — поступления, отрицательные — выплаты.

### Дисконтирование
Деньги во времени не равноценны: 1000 сейчас ≠ 1000 через год (можно положить под процент). Ставка $r$ — коэффициент пересчёта (депозит, стоимость капитала, альтернативные вложения).

### Приведённая стоимость (PV)
Один платёж: $PV = \dfrac{C}{(1 + r)^t}$.

Поток: $PV = \sum_{k=0}^n \dfrac{C_k}{(1+r)^{t_k}}$.

Будущая стоимость: $FV = \sum C_k (1+r)^{T - t_k}$.

При непрерывном начислении: $PV = \sum C_k e^{-r t_k}$ или $\int_0^T f(t) e^{-rt} dt$.

### NPV — Net Present Value
PV в контексте инвестпроекта, где $C_0 < 0$ (вложение): $$NPV = C_0 + \sum_{k=1}^n \frac{C_k}{(1+r)^k}.$$

**Критерий принятия:** $NPV > 0$ — проект выгоден.

### Ренты (аннуитеты)
Регулярный поток — одинаковые платежи $C$ через равные периоды.

**Постнумерандо** (платёж в конце периода — стандарт): $$PV = C \cdot a_{n, r}, \quad a_{n, r} = \frac{1 - (1+r)^{-n}}{r}.$$

$$FV = C \cdot s_{n, r}, \quad s_{n, r} = \frac{(1+r)^n - 1}{r}.$$

**Пренумерандо** (платёж в начале периода): $PV = C \cdot a_{n, r} \cdot (1+r)$. Все платежи на период раньше → дороже в PV.

**Бессрочная рента** ($n \to \infty$, $r > 0$): $PV = C/r$.

**Растущая (Гордон)** при $r > g$: $PV = C/(r - g)$. Используется в модели дисконтирования дивидендов.

### Принцип эквивалентности

Два потока **эквивалентны**, если их приведённые стоимости равны при заданной ставке.

Используется для:

- **Замены** одной ренты на другую с изменёнными параметрами.
- **Объединения** нескольких рент в одну.
- **Реструктуризации** кредитов и пенсионных схем.

### IRR — внутренняя норма доходности

Ставка $r^*$, при которой $NPV = 0$: $$\sum_{k=0}^n \frac{C_k}{(1 + r^*)^k} = 0$$

**Содержательно:** встроенная доходность проекта.

**Критерий принятия:** $IRR > $ стоимости капитала.

**Единственность:** если знак $C_k$ меняется **один раз** ($C_0 < 0$, далее $\ge 0$) — единственный положительный IRR (правило Декарта). Несколько смен знаков → может быть несколько IRR или ни одного.

**MIRR** (модифицированный IRR) для нестандартных потоков — задаются явные ставки реинвестирования и заимствования.

### NPV vs IRR

Для **одного проекта** "брать/не брать" — IRR работает.

Для **сравнения взаимоисключающих проектов** — лучше **NPV**: IRR не учитывает масштаб (30% от 100 = 30, 20% от 1000 = 200 — IRR говорит A лучше, NPV правильно показывает B).

### Другие критерии

- **PI (Profitability Index):** $PV(\text{доходов}) / |C_0|$. Принимаем при $PI > 1$.
- **Payback Period** — срок окупаемости. Простой (без дисконтирования) или дисконтированный.

### Подводные камни

- IRR неявно предполагает реинвестирование по той же ставке — нереалистично для высоких IRR.
- Ставка $r$ должна соответствовать риску (CAPM, WACC).
- Бессрочная формула $C/r$ не работает при $r = 0$.
- Модель Гордона $C/(r-g)$ требует $r > g$, иначе бессмыслица.
- Несколько IRR при многократной смене знака — IRR теряет смысл.

### Пример
$C_0 = -1000$, $C_1 = 400$, $C_2 = 500$, $C_3 = 300$, $r = 10\%$.
$$NPV = -1000 + \frac{400}{1{,}1} + \frac{500}{1{,}21} + \frac{300}{1{,}331} \approx 2{,}2$$
$NPV > 0$, проект слабо выгоден. $IRR \approx 10{,}1\%$ — едва превышает ставку.
