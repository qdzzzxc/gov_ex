# Задача 6. Решающее дерево по кредитным историям (Семейное положение, Образование, Недвижимость, Доход, Профессия).

## Условие
Имеется набор данных по кредитным историям в банке (10 договоров).

| № | Семейное положение | Образование | Недвижимость | Наличие дохода | Профессия       | Кредит погашен |
|---|--------------------|-------------|--------------|----------------|-----------------|----------------|
| 1 | в браке            | высшее      | есть         | есть           | менеджер        | погашен        |
| 2 | в браке            | высшее      | нет          | нет            | продавец        | не погашен     |
| 3 | не в браке         | среднее     | нет          | нет            | инженер         | не погашен     |
| 4 | в браке            | среднее     | нет          | нет            | продавец        | погашен        |
| 5 | не в браке         | среднее     | есть         | нет            | менеджер        | не погашен     |
| 6 | не в браке         | высшее      | в ипотеке    | есть           | предприниматель | погашен        |
| 7 | не в браке         | среднее     | нет          | есть           | инженер         | погашен        |
| 8 | не в браке         | высшее      | в ипотеке    | нет            | продавец        | не погашен     |
| 9 | не в браке         | высшее      | нет          | нет            | инженер         | не погашен     |
| 10| в браке            | высшее      | есть         | нет            | предприниматель | не погашен     |

**Задачи:**
1. рассчитать энтропию столбца «Возраст»;
2. построить диаграмму решающего дерева;
3. выбрать, обосновать и реализовать кодирование категориальных переменных.

## Решение

### Идея
- Энтропия категориального признака:
$$H(X) = -\sum_{i} p_i \log_2 p_i,\qquad p_i = \frac{n_i}{n}.$$
- Информативность признака $A$ относительно метки $y$ (ID3):
$$\mathrm{IG}(y, A) = H(y) - \sum_{v} \frac{|D_v|}{|D|}\, H(y \mid A = v).$$
В каждой вершине жадно выбираем признак с максимальным $\mathrm{IG}$, пока выборка в листе не станет чистой или не закончатся признаки.

### 1) Про «энтропию столбца Возраст»
В данной таблице **столбца «Возраст» нет** — это явная опечатка методички (формулировка скопирована из соседней задачи). Поэтому посчитаем энтропию реально присутствующих столбцов:

$$H(\text{Семья}) = -\tfrac{4}{10}\log_2\tfrac{4}{10} - \tfrac{6}{10}\log_2\tfrac{6}{10} \approx 0{,}971.$$

$$H(\text{Образование}) = -\tfrac{6}{10}\log_2\tfrac{6}{10} - \tfrac{4}{10}\log_2\tfrac{4}{10} \approx 0{,}971.$$

$$H(\text{Недвижимость}) = -\tfrac{5}{10}\log_2\tfrac{5}{10} - \tfrac{3}{10}\log_2\tfrac{3}{10} - \tfrac{2}{10}\log_2\tfrac{2}{10} \approx 1{,}486.$$

$$H(\text{Доход}) = -\tfrac{3}{10}\log_2\tfrac{3}{10} - \tfrac{7}{10}\log_2\tfrac{7}{10} \approx 0{,}881.$$

$$H(\text{Профессия}) = -\sum_{k}\tfrac{n_k}{10}\log_2\tfrac{n_k}{10},\quad n=(2,3,3,2)\ \text{для (менеджер, продавец, инженер, предприниматель)} \approx 1{,}971.$$

$$H(y) = -\tfrac{4}{10}\log_2\tfrac{4}{10} - \tfrac{6}{10}\log_2\tfrac{6}{10} \approx 0{,}971.$$

Максимум энтропии при 4 равновероятных категориях — $\log_2 4 = 2$; «Профессия» с $H \approx 1{,}971$ почти равномерна.

### 2) Построение дерева (ID3)

**Корень.** Информативности относительно $y$:

| Признак       | IG     |
|---------------|--------|
| Семья         | 0,020  |
| Образование   | 0,020  |
| Недвижимость  | 0,010  |
| **Доход**     | **0,557** |
| Профессия     | 0,020  |

Корень — **Наличие дохода**.

**Ветви по Доход:**
- *есть*: $\{1, 6, 7\}$ — все «погашен» (лист).
- *нет*: $\{2, 3, 4, 5, 8, 9, 10\}$ — смесь, делим дальше.

**Подузел Доход = нет** ($H = -\tfrac{2}{7}\log_2\tfrac{2}{7} - \tfrac{5}{7}\log_2\tfrac{5}{7} \approx 0{,}863$):

| Признак      | IG     |
|--------------|--------|
| **Семья**    | **0,198** |
| Образование  | 0,198  |
| Недвижимость | 0,128  |
| Профессия    | 0,198  |

Берём **Семейное положение** (несколько признаков дают одинаковый IG; выбираем «Семья» как первый по порядку):
- *не в браке*: $\{3, 5, 8, 9\}$ — все «не погашен» (лист).
- *в браке*: $\{2, 4, 10\}$ — смесь $\{$не погашен, погашен, не погашен$\}$, делим дальше.

**Подузел Доход = нет, Семья = в браке** ($H \approx 0{,}918$):

| Признак      | IG     |
|--------------|--------|
| **Образование** | **0,918** |
| Недвижимость | 0,252  |
| Профессия    | 0,252  |

Делим по **Образованию**:
- *высшее*: $\{2, 10\}$ — оба «не погашен» (лист).
- *среднее*: $\{4\}$ — «погашен» (лист).

**Итоговое дерево:**
```
                       Доход
                ┌────────┴────────┐
              есть               нет
               │                  │
            погашен              Семья
            {1,6,7}      ┌────────┴────────┐
                      не в браке         в браке
                          │                  │
                       не погашен        Образование
                       {3,5,8,9}      ┌─────┴─────┐
                                   высшее      среднее
                                      │           │
                                  не погашен   погашен
                                   {2,10}        {4}
```

Глубина дерева — 3, все 10 объектов классифицируются правильно (точность на обучении $100\,\%$).

### 3) Кодирование категориальных переменных

| Столбец            | Тип                       | Порядок? | Способ                                                    |
|--------------------|---------------------------|----------|-----------------------------------------------------------|
| Семейное положение | бинарный                  | нет      | label / one-hot эквивалентны: `не в браке→0, в браке→1`   |
| Образование        | порядковый                | да (`среднее < высшее`) | ordinal: `среднее→0, высшее→1`              |
| Недвижимость       | 3 категории, слабый порядок | спорно | основной выбор — **one-hot** (`нет / есть / в ипотеке`); как ordinal по «доступу к ликвидности» можно `нет→0, в ипотеке→1, есть→2` |
| Наличие дохода     | бинарный                  | да (`нет < есть`) | бинарное `нет→0, есть→1`                            |
| Профессия          | номинальный (4 категории) | нет      | **one-hot** (4 индикатора): менеджер / продавец / инженер / предприниматель |
| Кредит погашен (y) | бинарный                  | —        | `не погашен→0, погашен→1`                                 |

**Обоснование.** «Профессия» не имеет естественного порядка — label-encoding ввёл бы фиктивную упорядоченность («инженер $<$ менеджер $<$ предприниматель $<$ продавец»), которую линейные/дистанционные модели восприняли бы как информативную. Для «Образования» порядок есть, ordinal-кодирование экономит размерность. «Недвижимость» — пограничный случай: для деревьев one-hot безопаснее (любая категория может стать узлом); для линейных моделей удобнее ordinal по убыванию обременений (нет $<$ в ипотеке $<$ есть). Бинарные признаки кодируются одной 0/1-колонкой.

Для самого ID3 кодирование не влияет на структуру дерева (сравниваются дискретные значения), но необходимо для `sklearn`/линейных моделей.

### Код
```python
import math
import pandas as pd
from collections import Counter

data = [
    ("в браке","высшее","есть","есть","менеджер","погашен"),
    ("в браке","высшее","нет","нет","продавец","не погашен"),
    ("не в браке","среднее","нет","нет","инженер","не погашен"),
    ("в браке","среднее","нет","нет","продавец","погашен"),
    ("не в браке","среднее","есть","нет","менеджер","не погашен"),
    ("не в браке","высшее","в ипотеке","есть","предприниматель","погашен"),
    ("не в браке","среднее","нет","есть","инженер","погашен"),
    ("не в браке","высшее","в ипотеке","нет","продавец","не погашен"),
    ("не в браке","высшее","нет","нет","инженер","не погашен"),
    ("в браке","высшее","есть","нет","предприниматель","не погашен"),
]
cols = ["Семья","Образование","Недвижимость","Доход","Профессия","y"]
df = pd.DataFrame(data, columns=cols)

def H(s):
    n = len(s); c = Counter(s)
    return -sum(k/n*math.log2(k/n) for k in c.values() if k)

def IG(d, col):
    n = len(d)
    return H(d.y) - sum(len(s)/n*H(s.y) for _,s in d.groupby(col, observed=True))

for c in cols:
    print(f"H({c}) = {H(df[c]):.4f}")
print()
for c in cols[:-1]:
    print(f"IG({c}) = {IG(df, c):.4f}")
```


### Ответ
1. Столбца «Возраст» в наборе нет (опечатка). Энтропии имеющихся столбцов: $H(\text{Семья}) = H(\text{Образование}) = H(y) \approx 0{,}971$, $H(\text{Доход}) \approx 0{,}881$, $H(\text{Недвижимость}) \approx 1{,}486$, $H(\text{Профессия}) \approx 1{,}971$.
2. Дерево: корень — **Наличие дохода**; при «нет» — деление по **Семейному положению**; при «в браке» — далее по **Образованию**. Глубина 3, точность на обучении $100\,\%$.
3. Ordinal — для упорядоченных (Образование, Доход), one-hot — для номинальной «Профессия», бинарное 0/1 — для «Семья», для «Недвижимости» предпочтительнее one-hot.

### Замечания
- В формулировке методички — опечатка («энтропия Возраста»), такого столбца здесь нет; в [[05 Решающее дерево кредитные истории Дети Стаж Доход Возраст]] он реально есть, формулировка явно скопирована оттуда.
- На корне четыре признака дают одинаковый $\mathrm{IG} = 0{,}020$ — рангование между ними условно; правило «брать первый по индексу» — стандартное соглашение ID3.
- При $n=10$ дерево гарантированно переобучается; для устойчивости — прунинг, `max_depth`, валидация.
