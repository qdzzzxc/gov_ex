# Задача 5. Решающее дерево по кредитным историям (Дети, Стаж, Доход, Возраст).

## Условие
Имеется набор данных по кредитным историям в банке (10 договоров).

| № | Наличие детей | Стаж работы | Средний доход | Возраст | Кредит погашен |
|---|---------------|-------------|---------------|---------|----------------|
| 1 | нет           | $\le 2$     | 41–70         | 46–60   | погашен        |
| 2 | нет           | $>2$        | 100–130       | 25–39   | не погашен     |
| 3 | нет           | $>2$        | 41–70         | 46–60   | погашен        |
| 4 | есть          | $\le 2$     | 71–100        | 25–39   | погашен        |
| 5 | есть          | $>2$        | $>130$        | 25–39   | не погашен     |
| 6 | нет           | $\le 2$     | 41–70         | 46–60   | погашен        |
| 7 | нет           | $>2$        | 41–70         | 18–24   | не погашен     |
| 8 | нет           | $\le 2$     | 41–70         | 46–60   | погашен        |
| 9 | есть          | $>2$        | $>130$        | 25–39   | не погашен     |
| 10| есть          | $>2$        | 71–100        | 25–39   | не погашен     |

**Задачи:**
1. рассчитать энтропию столбца «Возраст»;
2. построить диаграмму решающего дерева;
3. выбрать, обосновать и реализовать кодирование категориальных переменных.

## Решение

### Программа минимум

```python
import math, pandas as pd

# Таблица из условия: признаки + целевой ответ y.
data = [
    ("нет","<=2","41-70","46-60","погашен"),
    ("нет",">2","100-130","25-39","не погашен"),
    ("нет",">2","41-70","46-60","погашен"),
    ("есть","<=2","71-100","25-39","погашен"),
    ("есть",">2",">130","25-39","не погашен"),
    ("нет","<=2","41-70","46-60","погашен"),
    ("нет",">2","41-70","18-24","не погашен"),
    ("нет","<=2","41-70","46-60","погашен"),
    ("есть",">2",">130","25-39","не погашен"),
    ("есть",">2","71-100","25-39","не погашен"),
]
df = pd.DataFrame(data, columns=["Дети","Стаж","Доход","Возраст","y"])

def H(s):
    p = s.value_counts(normalize=True)                 # частоты категорий
    return -sum(pi * math.log2(pi) for pi in p)        # энтропия

def IG(col, d=df):
    n = len(d)                                         # информационный выигрыш признака
    return H(d.y) - sum(len(g)/n * H(g.y) for _, g in d.groupby(col))

print("H(Возраст) =", round(H(df["Возраст"]), 3))
print({c: round(IG(c), 3) for c in ["Дети","Стаж","Доход","Возраст"]})
print("Дерево: Возраст -> 18-24: не погашен; 46-60: погашен; 25-39 -> Стаж <=2: погашен, >2: не погашен")
```

### Полное решение

### Идея
- Энтропия столбца $X$ с категориальными значениями $\{x_1,\dots,x_k\}$ — мера неопределённости распределения:
$$H(X) = -\sum_{i=1}^k p_i \log_2 p_i,\qquad p_i = \frac{n_i}{n}.$$
- Алгоритм построения дерева — **ID3** (жадно по информативности признака):
$$\mathrm{IG}(X, A) = H(X) - \sum_{v} \frac{|X_v|}{|X|} H(X_v).$$
В качестве корня и каждого внутреннего узла берём признак с наибольшим $\mathrm{IG}$ (равносильно — наименьшим $H_{\text{условн}}$).

### 1) Энтропия столбца «Возраст»
Значения и частоты: 18–24 — 1; 25–39 — 5; 46–60 — 4.

$$H(\text{Возраст}) = -\tfrac{1}{10}\log_2\tfrac{1}{10} - \tfrac{5}{10}\log_2\tfrac{5}{10} - \tfrac{4}{10}\log_2\tfrac{4}{10} \approx 0{,}332 + 0{,}500 + 0{,}529 = \mathbf{1{,}361}.$$

Для сравнения, $H(y) = -0{,}5\log_2 0{,}5 - 0{,}5\log_2 0{,}5 = 1$ (классы погашен/не погашен идеально сбалансированы).

### 2) Построение дерева (ID3)

**Корень.** $\mathrm{IG}$ по всем признакам относительно целевой $y$:

| Признак | IG     |
|---------|--------|
| Дети    | 0,125  |
| Стаж    | 0,610  |
| Доход   | 0,439  |
| **Возраст** | **0,639** |

Корень — **Возраст**.

**Ветви по Возраст:**
- 18–24: $\{7\} \to$ не погашен (лист).
- 46–60: $\{1, 3, 6, 8\} \to$ все погашен (лист).
- 25–39: $\{2, 4, 5, 9, 10\}$ — смесь, делим дальше.

**Подузел 25–39.** Подсчёт $\mathrm{IG}$:

| Признак | IG     |
|---------|--------|
| Дети    | 0,073  |
| **Стаж** | **0,722** |
| Доход   | 0,322  |

Делим по **Стаж**:
- $\le 2$: $\{4\} \to$ погашен (лист).
- $>2$: $\{2, 5, 9, 10\} \to$ все не погашен (лист).

**Итоговое дерево:**
```
                     Возраст
            ┌───────────┼───────────┐
        18-24       25-39        46-60
          │           │             │
       не погашен   Стаж         погашен
                   ┌──┴──┐
                 <=2    >2
                  │      │
              погашен  не погашен
```

Все 10 объектов классифицируются правильно — точность на обучении 100 %.

### 3) Кодирование категориальных переменных

| Столбец | Тип       | Порядок?              | Способ                                  |
|---------|-----------|------------------------|-----------------------------------------|
| Дети    | бинарный  | нет (это «да/нет»)    | `нет→0, есть→1` (label / 1-hot эквивалентны) |
| Стаж    | бинарный, порядковый | да (`<=2 < >2`) | ordinal: `<=2→0, >2→1` |
| Доход   | порядковый (интервалы) | да (`41-70 < 71-100 < 100-130 < >130`) | ordinal: `0,1,2,3` |
| Возраст | порядковый (интервалы) | да (`18-24 < 25-39 < 46-60`)         | ordinal: `0,1,2` |
| Кредит погашен (y) | бинарный | — | `погашен→1, не погашен→0` |

**Обоснование:** для столбцов с естественным порядком (Стаж, Доход, Возраст) используем порядковое кодирование — оно сохраняет монотонность и удобнее для линейных моделей. Для деревьев способ кодирования не критичен, но ordinal-схема позволяет дереву ставить разбиения вида $\text{Доход} \le 1$.

Если бы признаки были непорядковые (например, «Профессия»), нужно было бы one-hot — иначе вводится ложный порядок.

### Код
```python
import math, pandas as pd
from collections import Counter

data = [
    ("нет","<=2","41-70","46-60","погашен"),
    ("нет",">2","100-130","25-39","не погашен"),
    ("нет",">2","41-70","46-60","погашен"),
    ("есть","<=2","71-100","25-39","погашен"),
    ("есть",">2",">130","25-39","не погашен"),
    ("нет","<=2","41-70","46-60","погашен"),
    ("нет",">2","41-70","18-24","не погашен"),
    ("нет","<=2","41-70","46-60","погашен"),
    ("есть",">2",">130","25-39","не погашен"),
    ("есть",">2","71-100","25-39","не погашен"),
]
df = pd.DataFrame(data, columns=["Дети","Стаж","Доход","Возраст","y"])

def H(s):
    n = len(s); c = Counter(s)
    return -sum(k/n*math.log2(k/n) for k in c.values() if k)

def IG(d, col):
    n = len(d)
    return H(d.y) - sum(len(s)/n*H(s.y) for _,s in d.groupby(col, observed=True))

print("H(Возраст) =", round(H(df.Возраст), 3))
for c in ["Дети","Стаж","Доход","Возраст"]:
    print(f"IG({c}) = {IG(df,c):.3f}")
```

### Прогон
```
H(Возраст) = 1.361
IG(Дети) = 0.125
IG(Стаж) = 0.610
IG(Доход) = 0.439
IG(Возраст) = 0.639
```

### Ответ
1. $H(\text{Возраст}) \approx 1{,}361$ бит.
2. Дерево: корень — Возраст; ветка 25–39 делится по Стажу; остальные две ветки — листы. Все 10 объектов классифицируются корректно.
3. Все категориальные признаки имеют естественный порядок, кроме «Дети» — для них используем порядковое кодирование, для «Дети» — бинарное.

### Замечания
- При $n=10$ дерево гарантированно переобучается; на практике нужны прунинг / `max_depth` / `min_samples_leaf` и валидация.
- В методичке встречается опечатка «энтропия Профессии» — такого столбца здесь нет, поэтому считаем энтропию имеющегося в таблице столбца Возраст.
- Если приоритет — интерпретация, оставляем ID3; если устойчивость — переходим на CART (Gini, бинарные разбиения).
