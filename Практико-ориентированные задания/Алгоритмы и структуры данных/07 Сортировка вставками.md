# Задача 7. Сортировка вставками.

## Условие
Реализуйте алгоритм сортировки вставками на языке программирования по вашему выбору. Покажите его применимость на массивах разной длины и оцените временную сложность для разных случаев.

## Решение

### Программа минимум

```python
def insertion_sort(a):
    a = list(a)                         # копия, чтобы не менять исходный массив
    for i in range(1, len(a)):          # слева уже отсортированная часть
        x, j = a[i], i - 1
        while j >= 0 and a[j] > x:      # сдвигаем большие элементы вправо
            a[j+1] = a[j]; j -= 1
        a[j+1] = x                      # вставляем элемент на место
    return a
```

### Полное решение

### Идея
На каждой итерации $i$ массив $a[0..i-1]$ уже отсортирован. Берём $key = a[i]$ и сдвигаем элементы $a[j] > key$ на одну позицию вправо, пока не найдём место. Сравнения и сдвиги — пропорциональны числу инверсий.

### Код

```python
def insertion_sort(a):
    a = list(a)
    for i in range(1, len(a)):
        key, j = a[i], i - 1
        while j >= 0 and a[j] > key:
            a[j+1] = a[j]
            j -= 1
        a[j+1] = key
    return a
```

### Прогон (время на массивах разной длины и формы)

| $n$  | случайный | возрастающий | убывающий |
|------|-----------|--------------|-----------|
| 100  | 0,12 мс   | 0,01 мс      | 0,24 мс   |
| 500  | 2,84 мс   | 0,03 мс      | 5,34 мс   |
| 2000 | 49,20 мс  | 0,13 мс      | 92,88 мс  |
| 5000 | 323,82 мс | 0,30 мс      | 623,20 мс |

Время растёт как $n^2$ для случайного и убывающего, и почти линейно для уже отсортированного.

### Сложности
- **Лучший случай (отсортированный):** $\Theta(n)$ — внутренний `while` срабатывает 0 раз.
- **Худший случай (обратный порядок):** $\Theta(n^2)$ — на каждой итерации $i$ сдвигов.
- **Средний случай:** $\Theta(n^2)$ (число инверсий случайной перестановки $\sim n^2/4$).
- Память: $O(1)$ дополнительной (in-place в исходнике; здесь копия для безопасности).
- **Стабильна** (одинаковые ключи сохраняют порядок) — потому что сдвигаем строго при `a[j] > key`.

### Замечания
- На малых $n$ ($\lesssim 30{-}50$) бьёт по константам merge / quick sort — поэтому используется как «дно рекурсии» в гибридных сортировках (Timsort, introsort).
- Хорошо подходит для **почти отсортированных** массивов с малым числом инверсий $I$: $O(n + I)$.
