# Задача 3. Стек.

## Условие
Реализуйте структуру данных «Стек» на языке программирования по вашему выбору. Необходимо реализовать создание пустого стека, добавление элемента, удаление элемента, отображение всех элементов стека. Желательно использовать ООП.

## Решение

### Программа минимум

```python
class Stack:
    def __init__(self): self.a = []        # пустой стек
    def push(self, x): self.a.append(x)    # положить на вершину
    def pop(self): return self.a.pop()     # снять с вершины
    def show(self): return self.a[::-1]    # вывести сверху вниз
```

### Полное решение

### Идея
LIFO: операции только с одним концом — вершиной. Реализация поверх динамического массива (`list`) даёт $O(1)$ амортизированно для всех операций.

### Код

```python
class Stack:
    def __init__(self):
        self._data = []

    def push(self, v): self._data.append(v)

    def pop(self):
        if not self._data: raise IndexError("pop from empty stack")
        return self._data.pop()

    def peek(self):
        if not self._data: raise IndexError("peek from empty stack")
        return self._data[-1]

    def __len__(self): return len(self._data)
    def __iter__(self): return reversed(self._data)  # сверху вниз

    def __repr__(self):
        return "Stack(top→bot)[" + ", ".join(map(repr, self)) + "]"
```

### Прогон
```
push 1,2,3 → Stack(top→bot)[3, 2, 1], len = 3
peek = 3
pop  = 3 → Stack(top→bot)[2, 1]
```

### Сложности
- `push`, `pop`, `peek`, `len` — $O(1)$ амортизированно.
- Итерация — $O(n)$ суммарно.

### Замечания
- Альтернатива — на основе односвязного списка (см. задачу 1): тогда `push`/`pop` — строго $O(1)$.
- `pop`/`peek` из пустого стека поднимают `IndexError`.
