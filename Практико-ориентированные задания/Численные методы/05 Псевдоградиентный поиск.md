# Задача 5. Псевдоградиентный поиск.

## Условие
Используя метод псевдоградиентного поиска, выполните 5 итераций поиска весов $a$ в уравнении $y = a_0 + a_1 x$ с шагом $\mathrm{d}a = 0{,}1$, если известный $y = \{5,\ 3,\ 8,\ 10,\ -4\}$, а $x = \{0,\ 2,\ 11,\ 6,\ -3\}$.

## Решение

### Программа минимум

```python
import numpy as np
x = np.array([0, 2, 11, 6, -3.]); y = np.array([5, 3, 8, 10, -4.])
a0 = a1 = 0.0; da = 0.1                         # начальные веса и шаг
for _ in range(5):
    r = y - a0 - a1*x                            # ошибки модели
    a0 += da*np.sign(r.mean())                   # шаг против знака градиента
    a1 += da*np.sign((r*x).mean())
print(a0, a1)
```

### Полное решение

### Идея
Минимизируется функционал MSE
$$L(a_0, a_1) = \frac{1}{n} \sum_{i=1}^n (y_i - a_0 - a_1 x_i)^2.$$
Градиент:
$$\frac{\partial L}{\partial a_0} = -\frac{2}{n}\sum (y_i - a_0 - a_1 x_i),\qquad \frac{\partial L}{\partial a_1} = -\frac{2}{n}\sum x_i(y_i - a_0 - a_1 x_i).$$

Псевдоградиентный поиск: вместо градиента используем его **знак**, шаг $\mathrm{d}a$ фиксированный:
$$a^{(k+1)}_j = a^{(k)}_j - \mathrm{d}a \cdot \mathrm{sign}\!\left(\frac{\partial L}{\partial a_j}\right).$$

Стартуем с $a^{(0)} = (0, 0)$, делаем 5 итераций.

### Код

```python
import numpy as np
y  = np.array([ 5,  3,  8, 10, -4], float)
xs = np.array([ 0,  2, 11,  6, -3], float)

def loss(a0, a1): return ((y - a0 - a1*xs)**2).mean()
def grad(a0, a1):
    r = y - a0 - a1*xs
    return -2*r.mean(), -2*(r*xs).mean()

a0, a1, da = 0.0, 0.0, 0.1
for k in range(1, 6):
    g0, g1 = grad(a0, a1)
    a0 -= da * np.sign(g0)
    a1 -= da * np.sign(g1)
    print(f"k={k}: a=({a0:+.2f},{a1:+.2f})  L={loss(a0,a1):.3f}")
```

### Прогон

### Ответ
После 5 итераций: $a_0 = 0{,}5$, $a_1 = 0{,}5$, $L \approx 15{,}55$.

### Замечания
- МНК-оптимум: $a_0 \approx 1{,}82$, $a_1 \approx 0{,}80$, $L \approx 8{,}05$ — за 5 итераций псевдоградиент не успел дойти.
- Все 5 итераций знаки градиентов отрицательные ⇒ оба коэффициента «лезут вверх» с одинаковым шагом $0{,}1$. Это типично для начала пути от нуля.
- Псевдоградиент не использует величину градиента, поэтому шаг по «крутому» $a_1$ такой же, как по «пологому» $a_0$ — отсюда медленность. Для ускорения берут отдельные шаги по координатам или используют истинный градиент.
