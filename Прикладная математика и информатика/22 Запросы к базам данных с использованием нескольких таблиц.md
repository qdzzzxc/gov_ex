# Запросы к базам данных с использованием нескольких таблиц. Объединение и пересечение отношений. Соединения отношений — основные виды соединений, их характеристики.
Нормализованные данные хранятся в разных таблицах. Запросы по нескольким таблицам строятся через соединения (JOIN) и теоретико-множественные операции, соответствующие операциям реляционной алгебры: UNION ↔ объединение ($\cup$), INTERSECT ↔ пересечение ($\cap$), EXCEPT ↔ разность ($\setminus$), JOIN ↔ соединение ($\bowtie$).

## JOIN-ы
- INNER JOIN ... ON cond — только пары, удовлетворяющие условию.
- LEFT JOIN — все строки левой + совпадения справа (NULL, если нет).
- RIGHT JOIN — зеркально (редко используется).
- FULL OUTER JOIN — совпадения + строки только из левой + строки только из правой, несовпадения с NULL.
- CROSS JOIN — декартово произведение $n \times m$, без `ON`. Возникает случайно, если забыли условие; намеренно — для генерации всех комбинаций.
- SELF JOIN — таблица с самой собой по разным алиасам, для иерархий через `manager_id`.
- NATURAL JOIN — по одноимённым столбцам автоматически. Хрупок к изменениям схемы, лучше явный `ON`.
## Типы соединений
- Эквисоединение — условие через `=`.
- Тета-соединение — через любой предикат (`<`, `BETWEEN`).
## Множественные операции
- UNION — объединение с дедупликацией.
- UNION ALL — без дедупликации (быстрее).
- INTERSECT — пересечение.
- EXCEPT (или MINUS в Oracle) — разность. Требования: одинаковое число столбцов, совместимые типы.
## Подзапросы
- Скалярный — возвращает одно значение, используется в `WHERE` или `SELECT`.
- В FROM — производная таблица: `FROM (SELECT...) AS t`.
- IN / NOT IN — список значений.
- EXISTS / NOT EXISTS — проверка наличия.
- Коррелированный — ссылается на колонки внешнего запроса, выполняется для каждой строки внешнего. Позволяет условия «относительно текущей строки». EXISTS vs IN: содержательное различие в поведении с NULL. `NOT IN` с подзапросом, содержащим NULL, возвращает пусто из-за тернарной логики — всегда использовать `NOT EXISTS`.
## CTE
`WITH name AS (SELECT ...)` — именованный временный результат, можно переиспользовать в запросе. Рекурсивные CTE (`WITH RECURSIVE`) — для иерархий и графов.
