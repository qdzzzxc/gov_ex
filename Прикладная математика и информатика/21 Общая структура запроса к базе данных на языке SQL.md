# Общая структура запроса к базе данных на языке SQL. Основные блоки оператора SELECT. Последовательность выполнения блоков в группировками данных.

SQL — декларативный язык: описываешь, **что** нужно получить, а не **как**. Логический порядок выполнения определяет семантику запроса, физический выбирает оптимизатор (индексы, алгоритмы JOIN, перестановки) — лишь бы результат совпадал.
## Синтаксис SELECT
```sql
SELECT [DISTINCT] выражения [AS алиас]
FROM    таблицы / JOIN-ы
WHERE   условие
GROUP BY столбцы
HAVING  условие_над_группами
ORDER BY столбцы [ASC|DESC]
LIMIT   n [OFFSET m];
```
## Логический порядок выполнения
$\text{FROM/JOIN} \to \text{WHERE} \to \text{GROUP BY + агрегаты} \to \text{HAVING} \to \text{SELECT} \to \text{DISTINCT} \to \text{ORDER BY} \to \text{LIMIT}$
## Назначение блоков
- `SELECT` — какие столбцы и выражения возвращать. `DISTINCT` убирает дубли, `AS` — алиасы.
- `FROM` — источник данных: таблицы, JOIN-ы (INNER, LEFT/RIGHT/FULL OUTER, CROSS), подзапросы, CTE.
- `WHERE` — фильтр строк до группировки. Нельзя агрегаты и алиасы из SELECT.
- `GROUP BY` — группировка по значениям выражений.
- `HAVING` — фильтр групп после агрегирования. Можно агрегаты.
- `ORDER BY` — сортировка результата. Алиасы из SELECT доступны.
- `LIMIT n OFFSET m` — пагинация.
## Следствия из порядка
- Алиасы из `SELECT` недоступны в `WHERE` и `GROUP BY`, но доступны в `ORDER BY` (он логически после SELECT).
- Агрегаты нельзя в `WHERE` — они ещё не вычислены.
- Столбец, не входящий в `GROUP BY` и не агрегированный, не может стоять в `SELECT`.
- `LIMIT` без `ORDER BY` — порядок строк не гарантирован, результат может меняться между запусками.
## Агрегатные функции
Схлопывают много значений в одно: `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`. Без `GROUP BY` — по всему набору, с ним — по группам. `COUNT(*)` vs `COUNT(col)` vs `COUNT(DISTINCT col)`: все строки / непустые значения / уникальные непустые.
## Подзапросы
- Скалярный — возвращает одно значение, можно в `SELECT` или `WHERE`.
- В `FROM` (derived table) — как временная таблица.
- В `WHERE` через `IN`, `EXISTS`, `=` — для фильтрации по другому набору.
## NULL — тернарная логика
NULL — «неизвестное значение». Сравнение с NULL даёт UNKNOWN (не TRUE и не FALSE), строка не проходит фильтр. Проверка — `IS NULL` / `IS NOT NULL`, не `= NULL`.
