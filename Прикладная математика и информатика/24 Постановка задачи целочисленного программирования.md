# Постановка задачи целочисленного программирования. Метод Гомори. Метод ветвей и границ.

**Целочисленное программирование (ЦП):** $$\min c^\top x \quad \text{при} \quad Ax \le b, \quad x \in \mathbb{Z}^n.$$

То же, что LP, плюс условие $x \in \mathbb{Z}$.

**ЦП — NP-трудная.** Общего полиномиального алгоритма нет.

### LP-релаксация
Та же задача **без** условия целочисленности. Решается быстро (симплекс).

**Свойство:** даёт **границу** для ЦП:

- Минимизация: $z_{LP} \le z_{ЦП}$ — **нижняя** граница.
- Максимизация: $z_{LP} \ge z_{ЦП}$ — **верхняя** граница.

LP-оптимум обычно нецелый — лежит в **вершине** многогранника. Целочисленный оптимум — внутри решётки целых точек, чаще не в вершине.

**Округление LP не работает:** округлённая точка может быть недопустима или далека от оптимума.

### Метод Гомори (отсечений)

**Идея:** добавлять линейные неравенства, "срезающие" нецелые куски многогранника.

**Алгоритм:**

1. Решить LP.
2. Если решение целое — готово.
3. Иначе — построить **отсечение Гомори**: линейное неравенство, которое
    - **отсекает** текущую нецелую точку,
    - **сохраняет** все целые точки исходной задачи.
4. Добавить отсечение → решить новую LP → повторять.

**Построение:** из строки симплекс-таблицы с нецелой базисной переменной берутся **дробные части** коэффициентов. Получается неравенство $\sum {a_j} x_j \ge {b}$.

**Сходимость:** за конечное число шагов.

**На практике:** сам по себе медленный. Используется как часть **branch-and-cut**.

### Метод ветвей и границ (B&B)

**Главная идея:** перебираем целые точки умно, используя LP-границы для отбраковки заведомо неперспективных веток.

**Алгоритм:**

1. Решить LP-релаксацию корня.
2. Если решение целое — готово.
3. Иначе выбрать нецелую координату $x_i = \alpha$, **разветвить** на две подзадачи:
    - $x_i \le \lfloor \alpha \rfloor$.
    - $x_i \ge \lceil \alpha \rceil$.
4. Решить LP в каждой ветке.
5. Поддерживать текущий **рекорд** $z^*$ — лучшее найденное целое решение.
6. **Отсечь** ветку (не углубляться), если:
    - LP-релаксация недопустима (пусто).
    - LP-оптимум целый → обновить рекорд.
    - LP-граница хуже рекорда → отбраковать без перебора.
7. Иначе — ветвиться дальше.

**Дерево поиска:** корень = исходная задача, листья = целые решения или отсечённые ветки.

### Стратегии в B&B

**Выбор ветки для раскрытия:**

- best-first (с лучшей LP-границей).
- depth-first (экономия памяти).

**Выбор переменной для ветвления:**

- most-fractional (ближе всего к 0.5).
- strong branching (пробное решение обеих веток).

### Branch-and-cut

Современные решатели (CPLEX, Gurobi, CBC) комбинируют **B&B + отсечения Гомори** в каждом узле дерева. Главный практический алгоритм для ЦП.

### Сравнение

|Метод|Идея|На практике|
|---|---|---|
|**Гомори**|LP + последовательные отсечения|медленно сам по себе|
|**B&B**|Дерево подзадач + LP-границы для отсечения|стандарт|
|**Branch-and-cut**|B&B + отсечения Гомори в узлах|основа индустриальных решателей|

### Подводные камни

- LP-релаксация может быть слабой → B&B уходит в большое дерево.
- Хорошая формулировка модели (сильные неравенства, индикаторы) на порядки ускоряет.
- Симметрии в задаче → большое дерево. Нужно ломать симметрию.
- Бинарные задачи большой размерности → специальные эвристики или SAT.

### Главное на экзамене

1. **ЦП = LP + целочисленность.** NP-трудна.
2. **LP-релаксация** — граница для ЦП. Округление не работает.
3. **Гомори:** итеративное добавление линейных отсечений, сохраняющих все целые точки.
4. **B&B:** дерево подзадач с разветвлением по нецелой переменной. LP-границы отбраковывают ветки.
5. **Три случая отсечения ветки в B&B:** недопустимая LP, целое решение, граница хуже рекорда.
