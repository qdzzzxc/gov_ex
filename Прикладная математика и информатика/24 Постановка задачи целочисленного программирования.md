# Постановка задачи целочисленного программирования. Метод Гомори. Метод ветвей и границ.

**Целочисленное программирование (ЦП):** $$\min c^\top x \quad \text{при} \quad Ax \le b, \quad x \in \mathbb{Z}^n$$ То же, что LP, плюс условие $x \in \mathbb{Z}$. ЦП — **NP-трудная**, общего полиномиального алгоритма нет.
## LP-релаксация
Та же задача **без** условия целочисленности. Решается быстро (симплекс). Свойство: даёт **границу** для ЦП:
- Минимизация: $z_{LP} \le z_{ЦП}$ — нижняя граница.
- Максимизация: $z_{LP} \ge z_{ЦП}$ — верхняя граница. LP-оптимум обычно нецелый — лежит в вершине многогранника. Целочисленный оптимум — внутри решётки целых точек, чаще не в вершине. Округление LP не работает: округлённая точка может быть недопустима или далека от оптимума.
## Метод Гомори (отсечений)
Идея: добавлять линейные неравенства, «срезающие» нецелые куски многогранника. Алгоритм:
1. Решить LP.
2. Если решение целое — готово.
3. Иначе построить **отсечение Гомори**: линейное неравенство, которое **отсекает** текущую нецелую точку и **сохраняет** все целые точки исходной задачи.
4. Добавить отсечение → решить новую LP → повторять. Построение: из строки симплекс-таблицы с нецелой базисной переменной берутся дробные части коэффициентов, получается неравенство $\sum {a_j} x_j \ge {b}$. Сходимость за конечное число шагов. На практике сам по себе медленный, используется в составе branch-and-cut.
## Метод ветвей и границ (B&B)
Главная идея: перебираем целые точки умно, используя LP-границы для отбраковки заведомо неперспективных веток. Алгоритм:
1. Решить LP-релаксацию корня.
2. Если решение целое — готово.
3. Иначе выбрать нецелую координату $x_i = \alpha$, **разветвить** на две подзадачи: $x_i \le \lfloor \alpha \rfloor$ и $x_i \ge \lceil \alpha \rceil$.
4. Решить LP в каждой ветке.
5. Поддерживать текущий **рекорд** $z^*$ — лучшее найденное целое решение.
6. **Отсечь ветку** (не углубляться), если:
    - LP-релаксация недопустима (пусто);
    - LP-оптимум целый → обновить рекорд;
    - LP-граница хуже рекорда → отбраковать без перебора.
7. Иначе — ветвиться дальше. Дерево поиска: корень = исходная задача, листья = целые решения или отсечённые ветки. Современные решатели (CPLEX, Gurobi, CBC) комбинируют B&B с отсечениями Гомори в каждом узле дерева (**branch-and-cut**) — это основа индустриальных решателей ЦП.
