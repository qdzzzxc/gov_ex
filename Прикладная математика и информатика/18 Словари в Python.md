# Словари в Python. Назначение, особенности, создание, изменение. Основные операции и методы. Способы итерации по словарям.
## Назначение
`dict` — встроенный отображающий тип (mapping), ассоциативный массив пар ключ-значение. Обеспечивает доступ, вставку и удаление по ключу за O(1) в среднем.
## Реализация
Хеш-таблица с открытой адресацией поверх компактного представления: массив индексов + плотный массив записей в порядке вставки (с 3.6+). Сложность: O(1) в среднем, O(n) в худшем случае (массовые коллизии). При заполнении ~2/3 — ресайзинг с перехешированием. Хеш строк рандомизирован между запусками (защита от HashDoS).
## Особенности
- Ключи должны быть хешируемыми — требуются корректные `__hash__` и `__eq__` (равные объекты обязаны иметь равные хеши).
- Ключи уникальны, значения любые.
- Порядок вставки сохраняется (с 3.7+).
- Нельзя менять словарь во время итерации — `RuntimeError`.
## Создание
- Литерал: `{'a': 1, 'b': 2}`, пустой `{}`.
- Конструктор: `dict(a=1, b=2)`, `dict([('a', 1), ('b', 2)])`, `dict(other_dict)`.
- `dict.fromkeys(keys, value)` — все ключи с одним значением.
- Comprehension: `{k: v for k, v in pairs}`.
## Изменение
- `d[k] = v` — вставка/обновление.
- `del d[k]` — удаление.
- `d.update(other)` — массовое обновление из другого словаря или итерируемого.
- `d.pop(k, default)` — удаление с возвратом значения.
- `d.popitem()` — удаление и возврат последней пары (LIFO с 3.7+).
- `d.clear()` — очистка.
## Основные методы доступа
- `d[k]` — доступ, `KeyError` если ключа нет.
- `d.get(k, default)` — доступ без исключения.
- `d.setdefault(k, default)` — вернуть значение, вставив `default` если ключа нет.
- `k in d` — проверка наличия ключа за O(1).
## Итерация
- `for k in d` — по ключам (поведение по умолчанию).
- `for v in d.values()` — по значениям.
- `for k, v in d.items()` — по парам. `keys()`, `values()`, `items()` возвращают view-объекты — динамические представления, отражающие изменения словаря, а не копии.
