# Словари в Python. Назначение, особенности, создание, изменение. Основные операции и методы. Способы итерации по словарям.

## TL;DR

**Словарь** (`dict`) — встроенный тип Python для хранения пар `ключ: значение`. Он нужен, когда данные удобнее искать не по номеру позиции, а по смысловому ключу: имени, id, коду товара, слову, настройке.

Ключи должны быть **хешируемыми**: неизменяемыми или ведущими себя как неизменяемые относительно `__hash__` и `__eq__`. Типичные ключи: `str`, `int`, `float`, `bool`, `tuple` из хешируемых элементов. Списки, множества и обычные словари ключами быть не могут.

`dict` реализован как **хеш-таблица**. Чтение, вставка и удаление работают за $O(1)$ в среднем и $O(n)$ в худшем случае при большом числе коллизий. Начиная с Python 3.7 порядок вставки является гарантированной частью языка.

```python
ages = {"Ann": 20, "Bob": 22}
ages["Kate"] = 19          # добавление
ages["Ann"] = 21           # изменение
print(ages.get("Tom", 0))  # безопасное чтение
del ages["Bob"]            # удаление
```

## Развёрнуто

### Назначение словарей

Словарь задаёт отображение:
$$
key \mapsto value.
$$

Он подходит для задач, где нужен быстрый доступ по ключу:

- телефонная книга: `имя -> телефон`;
- частотный словарь: `слово -> количество`;
- кэш: `аргументы -> результат`;
- конфигурация: `название параметра -> значение`;
- группировка объектов по id, категории или дате.

```python
word_count = {}

for word in ["cat", "dog", "cat"]:
    word_count[word] = word_count.get(word, 0) + 1

print(word_count)  # {'cat': 2, 'dog': 1}
```

### Создание словаря

Основные способы:

```python
empty = {}

user = {"name": "Ann", "age": 20}

same_user = dict(name="Ann", age=20)

pairs = dict([("a", 1), ("b", 2)])

squares = {x: x * x for x in range(5)}
```

Ключ в одном словаре уникален. Если указать ключ повторно, останется последнее значение:

```python
d = {"x": 1, "x": 2}
print(d)  # {'x': 2}
```

### Чтение, изменение и удаление

Доступ по существующему ключу:

```python
user = {"name": "Ann", "age": 20}
print(user["name"])  # Ann
```

Если ключа нет, выражение `d[key]` вызывает `KeyError`. Для безопасного чтения используют `get`:

```python
print(user.get("city"))              # None
print(user.get("city", "Unknown"))   # Unknown
```

Добавление и изменение имеют одинаковый синтаксис:

```python
user["city"] = "Moscow"  # добавление
user["age"] = 21         # изменение
```

Удаление:

```python
age = user.pop("age")        # удаляет и возвращает значение
city = user.pop("city", "")  # не падает, если ключа нет

del user["name"]             # удаляет ключ, но KeyError, если ключа нет
user.clear()                 # очищает словарь
```

Проверка наличия ключа:

```python
if "name" in user:
    print(user["name"])
```

### Основные методы

| Метод | Назначение |
|---|---|
| `d.get(key, default)` | вернуть значение или `default`, если ключа нет |
| `d.setdefault(key, default)` | вернуть значение; если ключа нет, создать его с `default` |
| `d.update(other)` | добавить или перезаписать пары из другого словаря/итератора пар |
| `d.pop(key[, default])` | удалить ключ и вернуть значение |
| `d.popitem()` | удалить и вернуть последнюю добавленную пару |
| `d.keys()` | динамическое представление ключей |
| `d.values()` | динамическое представление значений |
| `d.items()` | динамическое представление пар `(ключ, значение)` |
| `d.copy()` | поверхностная копия |
| `dict.fromkeys(keys, value)` | создать словарь с заданными ключами и одним значением |

Пример с `setdefault` для группировки:

```python
groups = {}

for name, group in [("Ann", "A"), ("Bob", "B"), ("Kate", "A")]:
    groups.setdefault(group, []).append(name)

print(groups)  # {'A': ['Ann', 'Kate'], 'B': ['Bob']}
```

### Итерация

По умолчанию словарь итерируется по ключам:

```python
scores = {"Ann": 5, "Bob": 4}

for name in scores:
    print(name, scores[name])
```

Более явно:

```python
for key in scores.keys():
    print(key)

for value in scores.values():
    print(value)

for key, value in scores.items():
    print(key, value)
```

Представления `keys`, `values`, `items` не являются независимыми списками: они отражают текущее состояние словаря.

### Хешируемость ключей

Словарь ищет ключ через хеш:
$$
hash(key) \to \text{позиция в таблице}.
$$

Объект может быть ключом, если:

- для него определён стабильный хеш;
- равные объекты имеют равные хеши;
- его состояние, участвующее в сравнении и хешировании, не меняется во время нахождения в словаре.

```python
ok = {(1, 2): "point"}  # tuple из int хешируем

bad = {}
# bad[[1, 2]] = "point"  # TypeError: unhashable type: 'list'
```

Для пользовательских классов важно согласовывать `__eq__` и `__hash__`: если объекты сравниваются по содержимому, хеш должен строиться по тем же неизменяемым полям.

### Порядок вставки

В современных версиях Python порядок обхода словаря совпадает с порядком добавления ключей:

```python
d = {}
d["first"] = 1
d["second"] = 2
d["third"] = 3

print(list(d))  # ['first', 'second', 'third']
```

Если существующему ключу присвоить новое значение, его позиция в порядке вставки не меняется. Если ключ удалить и добавить заново, он окажется в конце.

### Сложности

| Операция | Средний случай | Худший случай |
|---|---:|---:|
| `d[key]` | $O(1)$ | $O(n)$ |
| `d[key] = value` | $O(1)$ | $O(n)$ |
| `key in d` | $O(1)$ | $O(n)$ |
| `del d[key]`, `pop` | $O(1)$ | $O(n)$ |
| обход всех элементов | $O(n)$ | $O(n)$ |
| `len(d)` | $O(1)$ | $O(1)$ |

Худший случай связан с коллизиями и деградацией хеш-таблицы. На практике встроенный `dict` оптимизирован, а хеш строк рандомизирован между запусками для защиты от атак вида HashDoS.

### Типичные ошибки

**1. Обращение к отсутствующему ключу через `[]`.**

```python
counts = {}
# counts["x"] += 1  # KeyError
counts["x"] = counts.get("x", 0) + 1
```

**2. Использование изменяемого объекта как ключа.**

```python
# d = {[1, 2]: "value"}  # TypeError
d = {(1, 2): "value"}
```

**3. Изменение размера словаря во время итерации.**

```python
d = {"a": 1, "b": 2, "c": 3}

for key in list(d):
    if d[key] % 2 == 1:
        del d[key]
```

**4. Опасный общий список в `fromkeys`.**

```python
bad = dict.fromkeys(["a", "b"], [])
bad["a"].append(1)
print(bad)  # {'a': [1], 'b': [1]}

good = {key: [] for key in ["a", "b"]}
```

**5. Путаница между ключами и значениями.**

```python
d = {"a": 1}

print("a" in d)        # True: проверяются ключи
print(1 in d.values()) # True: проверяются значения
```
